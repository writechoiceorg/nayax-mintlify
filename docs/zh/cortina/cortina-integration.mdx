Cortina 的集成流程包含多个阶段,以确保解决方案在上线前得到实施和测试。下图展示了集成流程。

</Frame>

该过程的步骤如下：

1. 启动会议:集成方与 Nayax 团队举行初次会议,以统一预期、确定高层目标,并介绍集成方项目的相关方、技术团队、Nayax 集成工程师以及产品团队。
2. **范围界定**：集成商填写一份详细的入职表格，明确定义集成要求、用例、限制以及任何必要的项目依赖项。
3. **测试环境搭建**：Nayax 集成工程师根据范围界定表建立内部设置。他们配置任何所需的网络，例如 VPN、测试设备以及 Nayax Core 上的相应机器。
4. **开发阶段**：集成商开始实施 Cortina API。Nayax 集成工程师提供咨询并解决可能出现的任何技术疑问或问题。
5. 测试与认证:集成方通过内部测试验证其解决方案。随后集成方需完成强制性的 Cortina 认证测试,以确保解决方案实施正确。集成方还必须完成所有所需的其他认证,例如 EMV L3。
6. 生产环境配置:认证通过后,Nayax 集成工程师会在 Nayax 生产服务器上配置已批准的集成,并在生产环境中完成所有最终的网络要求。此时,集成过程中使用的令牌会被替换为新的生产令牌。
7. **试点阶段**：项目将启动一个试点阶段，使用有限数量的活跃​​设备来验证实际性能。该试点阶段将持续约定的时间或交易量，集成商必须解决在此阶段发现的任何问题。
8. **最终确定**：一旦双方确认所有问题均已解决且试点结果令人满意，项目即进入后续支持阶段。双方将共享各自生产支持团队、网络运营中心 (NOC) 和运维团队的联系方式。
