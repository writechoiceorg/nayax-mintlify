Cortina 使用 `SaleEndNotification` 命令通知集成方有关 Cortina 支付提供商的交易请求已完成的信息。

## 要求

Nayax 的服务器会向您配置的 `SaleEndNotification` 端点发送 POST 请求,其格式如下:

`(customer's URL)/Cortina/SaleEndNotification`

有效载荷包含以下详细信息：

```json
{
  "BasicInfo": {
    "TransactionId": "N99O4XURDQYJZIK4OTCAWF1RHHA40TZWVCRB",
    "NayaxTransactionId": 3584697384,
    "Amount": 5,
    "CurrencyCode": "ILS",
    "CurrencyNumeric": "376",
    "SiteId": 1,
    "MachineAuTime": "071221132914",
    "TimeoutMS": 15000,
    "IsProductSelected": true,
    "PaymentMethodID": 1,
    "BillingProviderId": 1,
    "AuditNumber": 342,
    "NayaxRRN": "111111",
    "IsDeferredAuthorization": true,
    "AgeLimitation": 18,
    "IsSoftDeclineCompletion": true,
    "TransitElement": {
      "TransitTransactionType": 1,
      "DenyListIndication": 0
    }
  },
  "PaymentInfo": {
    "SrvTranId": "3584697388",
    "AuthCode": "333333",
    "AuthAmount": 5.5,
    "SettAmount": 5.5,
    "RRN": "111111",
    "Token": "string",
    "AuthDateTime": "071221133300",
    "SettDateTime": "071221133330",
    "TraceNumber": "4444",
    "AuthSource": "string",
    "AdditionalData": "string",
    "IsGatewayTimeout": false,
    "ProviderExternalData": "string",
    "InitiateTranReference": 11223344,
    "IsRevalueCard": false
  },
  "DynamicURL": "string",
  "CustomData": {
    "DirectActor": "String",
    "Operator": "String",
    "Distributor": "String",
    "Actor": "String",
    "Machine": "String",
    "SoftDecline": "String"
  }
}
```

## 回复

客户支付方式将发送给 Nayax 服务器的 `SaleEndNotification` 响应。其负载应如下所示:

```json
{
  "Status": {
    "Verdict": "string",
    "Code": 0,
    "StatusMessage": "string",
    "CustomDeclineCode": "string"
  }
}
```

      <td>
        `TranIDCipher`
      </td>

      <td>
        String(36)
      </td>

      <td>
        是的
      </td>

      <td>
        集成方生成一个 36 个字符(即 288 位)的交易 ID。
        随后集成方附加字符 = 和 RandomNumber,以生成 64 个字符的密文。
        格式:TransactionId=RandomNumber
        集成方使用 AES ECB 模式,以 Secret Token 对密文进行加密。
        AES 密钥为与请求中提供的 TokenID 匹配的 Secret Token 的最后(最右侧)256 位(即 32 个字符)。
        Nayax 将使用 AES ECB 解密并提取交易 ID
        该交易 ID 值随后将用于所有后续请求。

        Information regarding how to generate the TranIDCipher (via encrypting the token alongside the Random Number and the customer's Transaction ID can be reviewed under [Start Session authentication process](/docs/cortina/start-session/cortina-auth#/)
      </td>
    </tr>

        3 - 持卡人超过每日/每周/每月交易次数限制

        <br />

        5 - 疑似欺诈

        <br />
