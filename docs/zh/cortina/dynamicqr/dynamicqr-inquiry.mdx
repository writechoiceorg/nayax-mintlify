Cortina DQR 使用 `Inquiry` 查询交易 ID 的批准状态。此调用将包含交易的最终金额以及在 `GenerateQR` 调用中生成的交易 ID。对此调用的“Approved”响应通常会尝试出售/释放产品。“Decline”将终止该交易,而“Pending”响应将在预定义的时间后触发后续的 `Inquiry` 请求。
Cortina 支付提供商应验证该信息,并相应地以待在 Nayax 设备上显示的动态 QR 信息作出响应。

该命令在整个授权流程期间都至关重要。

## 要求

Nayax 的服务器向您配置的 `Inquiry` 端点发送 POST 请求,格式如下:

`(customer's URL)/Cortina/DynamicQR/Inquiry`

## 回复

客户支付方式将发送给 Nayax 服务器的 `Inquiry` 响应。

通常响应会是“Approved”或带有代码 993(表示“pending”,即支付提供商仍在决定是否批准该消费者,等待“Approved”/“Declined”的最终响应)的“Declined”。在“pending”响应的情况下,Nayax 服务器将持续轮询,直到收到最终响应(“Approved”/“Declined”)或达到 Nayax Core 中配置的限制为止。

有效载荷应如下所示：

<CodeGroup>
  ```json JSON
  {
    "Status": {
      "Verdict": "Approved",
      "Code": 0,
      "StatusMessage": "Cortina V2 Stub Tester"
    },
    "PaymentInfo": {
      "AuthCode": "333333",
      "AuthAmount": 0.1,
      "SettAmount": 0.1,
      "Token": "Cortina V2 Stub Tester Token",
      "AuthDateTime": "230221101008",
      "SettDateTime": "230221101009",
      "TraceNumber": "4444",
      "AuthSource": "555555",
      "AdditionalData": ""
    }
  }
  ```

  ```json JSON-Pending
  {
    "Status": {
      "Verdict": "Declined",
      "Code": 993,
      "StatusMessage": "Pending"
    },
    "PaymentInfo": {
      "AuthCode": "333333",
      "AuthAmount": 0.1,
      "SettAmount": 0.1,
      "Token": "Cortina V2 Stub Tester Token",
      "AuthDateTime": "230221101008",
      "SettDateTime": "230221101009",
      "TraceNumber": "4444",
      "AuthSource": "555555",
      "AdditionalData": ""
    }
  }
  ```
</CodeGroup>
