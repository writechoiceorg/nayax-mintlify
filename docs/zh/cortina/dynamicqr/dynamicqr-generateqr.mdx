Cortina DQR 使用 `GenerateQR` 命令通知集成方有关消费者产品选择的信息,以便其发送 QR 生成详情。该命令包括交易 ID 标识(在 /StartSession 中生成)、所需金额、商户及机器详情(以及更多信息)。Cortina 支付提供商应验证该信息,并相应地以待在 Nayax 设备上显示的动态 QR 信息作出响应。

该命令对于进入授权流程至关重要。

## 要求

Nayax 的服务器向您配置的 `GenerateQR` 端点发送 POST 请求,格式如下:

`(customer's URL)/Cortina/DynamicQR/GenerateQR`

## 回复

客户支付方式将发送给 Nayax 服务器的 `GenerateQR` 响应。负载应如下所示:

```json
{
  "QRCode": "QR_URL_to_be_presented_on_the_device's_screen",
  "QRData": {
    "ChannelId": "19",
    "RRN": "111111",
    "SrvTranId": "236584335969834429321847829253667359"
  },
  "Status": {
    "Verdict": "Approved",
    "Code": 0,
    "StatusMessage": "Cortina V2 Stub Tester"
  },
  "additionalProperties": false
}
```
