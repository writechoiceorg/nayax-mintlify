Cortina SQR 使用 `Void` 命令通知集成方,以在下述任一情况发生时启动与 Cortina 支付提供商的取消流程:

1. 自动售货过程中出现的问题 - 消费者按下取消按钮/机器售货故障（产品分发/服务提供方面出现问题）。
2. 在超时之前不会收到 `Sale` 响应。

此命令对于让支付提供商知道交易已在 Nayax 端（以及从消费者的角度来看）被取消至关重要，并且执行此命令是为了让支付提供商在其端完成取消操作。

- 注意 - 此命令仅适用于 PreSelection。在 PreAuthorization 流程中,其等效命令为 `Cancel` 命令。

## 要求

Nayax's servers send a POST request to your configured `Void` endpoint, which would be of the following format:

`(customer's URL)/Cortina/StaticQR/Void`

该负载（payload）包含读卡操作的完整结果：

## 回复

客户的支付方式会向 Nayax 服务器发送的撤销响应。负载应如下所示：

```json
{
  "Status": {
    "Verdict": "Approved",
    "StatusMessage": "Cortina V2 Stub Tester"
  },
  "PaymentInfo": {
    "SrvTranId": "236584335969834429321847829253667359",

    "SettDateTime": "230221101009",
    "TraceNumber": "4444",
```
