Cortina SQR 使用 `Cancel` 命令通知集成方,以在下述任一情况发生时启动与 Cortina 支付提供商的取消流程:

1. 收到失败的 `Authorization` 响应(由于超时或 `Authorization` 响应中包含错误代码)。
2. 产品选择过程中出现的问题 - 消费者无操作/按下取消按钮/消费者选择了无效的产品。
3. 机器故障（产品分发/服务提供方面出现问题）

此命令对于让支付提供商知道交易已在 Nayax 端（以及从消费者的角度来看）被取消至关重要，并且执行此命令是为了让支付提供商在其端完成取消操作。

- 注意 - 此命令仅适用于 PreAuthorization。在 PreSelection 流程中,其等效命令为 `Void` 命令。

## 要求

Nayax 的服务器会向您配置的 `Cancel` 端点发送 POST 请求,格式如下:

`(customer's URL)/Cortina/StaticQR/Cancel`

## 回复

客户的支付方式发送给 Nayax 服务器的 Cancel 响应。有效负载应如下所示:

```json
{
  "Status": {
    "Verdict": "Approved",
    "StatusMessage": "Cortina V2 Stub Tester"
  }
}
```

### 响应参数
