Cortina SQR 使用 `Refund` 命令通知集成方退款请求,以启动与 Cortina 支付提供商的退款流程。该请求将通过以下任一方式触发:

- 使用 Nayax 的动态交易监控器，选择相关交易，右键单击，然后选择“请求退款”。
- Nayax 的 Lynx API - 通过退款命令

## 要求

Nayax's servers send a POST request to your configured `Refund` endpoint, which would be of the following format:

`(customer's URL)/Cortina/StaticQR/Refund`

## 回复

客户的支付方式会向 Nayax 服务器发送的退款响应。负载应如下所示：

```json
{
  "Status": {
    "Verdict": "Approved",
    "StatusMessage": "Cortina V2 Stub Tester"
  }
}
```

### 响应参数

下表描述了 API 响应中返回的字段：

<table>
  <thead>
    <tr>
      字段
      类型

  <tbody>
    <tr>

    </tr>

    <tr>
      `Verdict`
      String

      <td style={{ textAlign: "left" }}>
        表示**请求最终状态的**常量字符串：

        <ul>
          <li>**`Approved`**</li>
          <li>**`Declined`**</li>
        </ul>

    <tr>
      `Code`
      Int(32)

      <td style={{ textAlign: "left" }}>
        拒绝代码（Decline Code）。条件性字段——仅当 `Verdict` 为 “Declined” 时才会出现。有效代码包括：

        <ul>
          1：余额不足
          5：疑似欺诈
          6：系统一般性故障
          7：金额无效
          8 / 998：格式/解析错误或验证错误
          992：超时异常
          997：缺少必填参数
          999：一般性异常
          有关其他状态和内部错误（例如 40、990、1011），请参阅完整列表。
        </ul>
      </td>
    </tr>

    <tr>
      `StatusMessage`
      String(255)

      <td style={{ textAlign: "left" }}>
        包含**交易状态消息**或有关结果的其他不同数据的自由文本字段。

    </tr>

    </tr>

        </ul>
      </td>
    </tr>

    <tr>
      `RegularCredit`
      Double

      <td style={{ textAlign: "left" }}>
        根据 `RegularCreditType` 计算的常规信用额度值（金额或交易次数）。使用小数位数 2。
      </td>

    <tr>
      `RevalueCredit`
      Int(32)

      <td style={{ textAlign: "left" }}>
