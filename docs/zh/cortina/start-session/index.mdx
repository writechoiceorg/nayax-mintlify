Cortina 使用 `Start Session` 命令通知集成商,与 Cortina 支付提供商之间的交易请求已初始化。

<Note>
  重要提示:对于 Cortina Static QR(SQR),使用的是 `Start` 命令而非 `Start`Session,且 Start 是由 Cortina 支付提供商发送给 Nayax,而不是反过来(如前文所述)。
</Note>

More on that in [Static QR- Start](/docs/cortina/staticqr/static-qr-start#/).

## 要求

Nayax 的服务器会向您配置的 `StartSession` 端点发送 POST 请求,其格式如下:

`(customer's URL)/Cortina/StartSession`

有效载荷包含以下详细信息：

```json
{
  "TokenId": "123456",
  "RandomNumber": "123456789qwertyuioasdfghjkl",
  "DynamicURL": "string",
  "CustomData": {
    "DirectActor": "String",
    "Operator": "String",
    "Distributor": "String",
    "Actor": "String",
    "Machine": "String",
    "SoftDecline": "String"
  }
}
```

## 回复

客户支付方式将发送给 Nayax 服务器的 `StartSession` 响应。其负载应如下所示:

```json
{
  "TranIDCipher": "a0Qnxm4fWMskzFXiMivn8BDiQVSL6be/NXIICC9HBoAiry6DUdKYPQh/YS1G8nObE6/0o9N4MFuYA7CTAxAnphuNJwBEjgBzKhhgpJ5ggnw=",
  "Status": {
    "Verdict": "Approved",
    "Code": 0,
    "StatusMessage": "string",
    "CustomDeclineCode": "string"
  }
}
```

          AES 密钥:与 `TokenID` 匹配的 Secret Token 的最右侧 256 位(32 个字符)。
          Nayax 的角色:解密该密文以提取交易 ID,该交易 ID 随后用于所有后续请求。
