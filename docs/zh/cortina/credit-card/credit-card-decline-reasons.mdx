当交易被拒绝时，Cortina API 会返回一个特定的拒绝原因代码，说明付款未被批准的原因。这些代码对于集成商和商户理解拒绝原因并采取相应措施至关重要。第三方集成商负责根据所使用的支付方式，返回相应的响应，包括列表中对应的拒绝代码。

处理信用卡交易时使用以下拒绝原因代码：

| 代码   | 描述                        |
| ---- | ------------------------- |
| 1    | 请联系发卡行                    |
| 2    | 请参阅发卡机构的特殊条款。             |
| 3    | 无效商户                      |
| 4    | 没收卡片                      |
| 5    | 拒绝交易                      |
| 6    | 错误                        |
| 7    | 没收卡片，特殊情况                 |
| 8    | 核实身份后予以受理                 |
| 9    | 请求处理中                     |
| 10   | 部分金额已批准                   |
| 11   | 已批准（VIP）                  |
| 12   | 无效交易                      |
| 13   | 无效金额                      |
| 14   | 无效卡号（不存在该卡号）              |
| 15   | 无此发卡行                     |
| 16   | 已批准，更新磁道 3                |
| 17   | 客户取消                      |
| 18   | 客户争议                      |
| 19   | 请重新输入交易                   |
| 20   | 无效响应                      |
| 21   | 未采取任何操作                   |
| 22   | 疑似故障                      |
| 23   | 交易费用不可接受                  |
| 24   | 接收方不支持文件更新                |

| 56   | 无卡片记录                     |
| 57   | 不允许该持卡人进行此交易              |
| 58   | 不允许该终端进行此交易               |

| 60   | 受卡方请联系收单机构                |
