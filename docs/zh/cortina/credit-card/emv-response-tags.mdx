在处理 EMV 接触式信用卡交易时，在授权/销售响应中包含 EMV 响应标签至关重要，以确保发卡行、终端和芯片卡之间的无缝通信。

- 必填的 EMV 响应标签：
  - 标签 8A：授权响应代码
  - 标签 91：发卡行认证数据
- 可选的 EMV 响应标签：
  - 标签 71 和标签 72：发卡行脚本

这些标签携带由发卡行提供的指令（脚本），终端会将其发送回卡片。它们用于更新卡片参数或管理生命周期事件（例如，锁卡和更新限额）。

如果在接触式交易中未返回必填的 EMV 响应标签（8A 和 91），或返回不正确，可能会导致交易在第二次生成 AC 步骤（Second GenAC）中失败并被拒绝。如果交易因响应标签不正确而被拒绝，将发送 `ReasonCode` 为 17 或 18 的 Cortina Cancel/Void。

## 如何实施

针对每笔接触式交易响应，将标签值返回到 "EMVTags" 对象中的相应字段。

```json
"EMVTags": {
"Tag91": "A93381BFF8C5E9EC0012",
"Tag8A": "3030",
"Tag71": "860D8424000008D466284E10D1FF7C",
"Tag72": "860D8424000008D466284E10D1FF7C"}
```
