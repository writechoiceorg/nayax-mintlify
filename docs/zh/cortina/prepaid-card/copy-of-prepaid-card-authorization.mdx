Cortina Prepaid 使用 `Authorization` 命令通知刷卡结果,以启动与 Cortina 支付提供商的授权流程。

此命令对于进行交易确认或拒绝处理至关重要。

## 要求

Nayax 的服务器会向您配置的 `Authorization` 端点发送 POST 请求,格式如下:

`(customer's URL)/Cortina/PrePaid/Authorization`

```json
{
"Status": {
"Verdict": "Approved",
"Code": 0,
"StatusMessage": "string"
},
"PaymentInfo": {
"SrvTranId": 2.3658433596983443e+35,
"AuthCode": 333333,
"AuthAmount": 0.1,
"SettAmount": 0.1,
"RRN": 111111,
"Token": "string",
"AuthDateTime": 230221101008,
"SettDateTime": 230221101009,
"TraceNumber": 4444,
"AuthSource": 555555,
"AdditionalData": "string",
"IsGatewayTimeout": true
},
"Balance": {
"RegularCreditType": 0,
"RegularCredit": 0,
"RevalueCredit": 0
}
}
```
