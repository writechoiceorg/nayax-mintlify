Cortina 预付使用 `Settlement` 命令通知集成方设备端结算流程的结果(即产品已出售),以便与 Cortina 支付提供商发起结算流程。

此命令对于进行交易确认或拒绝处理至关重要。

## 要求

Nayax 的服务器会向您配置的 `Settlement` 端点发送 POST 请求,格式如下:

`(customer's URL)/Cortina/PrePaid/Settlement`

## 回复

客户的支付方式发送给 Nayax 服务器的 Settlement 响应。有效负载应如下所示:

```json
{
  "Status": {
    "Verdict": "Approved",
    "Code": 0,
    "StatusMessage": "string"
  },
  "Balance": {
    "RegularCreditType": 0,
    "RegularCredit": 0,
    "RevalueCredit": 0
  }
}
```
