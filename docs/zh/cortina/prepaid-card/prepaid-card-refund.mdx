Cortina 预付使用 `Refund` 命令通知集成方有关退款请求的信息,以便与 Cortina 支付提供商发起授权流程。该请求将通过以下任一选项触发:

- 使用 Nayax 的动态交易监控器，选择相关交易，右键单击，然后选择“请求退款”。
- Nayax 的 Lynx API - 通过退款命令

## 要求

Nayax's servers send a POST request to your configured `Refund` endpoint, which would be of the following format:

`(customer's URL)/Cortina/PrePaid/Refund`

## 回复

客户的支付方式会向 Nayax 服务器发送的退款响应。负载应如下所示：

```json
{
  "Status": {
    "Verdict": "Approved",
    "Code": 0,
    "StatusMessage": "string"
  },
  "Balance": {
    "RegularCreditType": 0,
    "RegularCredit": 0,
    "RevalueCredit": 0
  }
}
```

### 响应参数

下表描述了 API 响应中返回的字段：

<details>
  参考字段

          <ul>
            1: 余额不足
            5: 疑似欺诈
            6: 一般系统故障
            7: 金额无效
            8 / 998: 格式/解析错误或验证错误
            992: 超时异常
            997: 缺少必填参数
            999: 一般异常
