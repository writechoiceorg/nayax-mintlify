Nayax eCom SDK 对 Credential on File（COF，存档凭证）交易使用令牌化。当触发 COF 初始化时，eCom API 会向 eCom SDK 返回一个 Nayax 卡片令牌。该令牌随后会存储在 Nayax 的系统中，您的应用程序可将其用于后续的循环交易，因为它表示持卡人已授权商户在未来存储并扣款其卡片。

令牌提供了一种安全的方法来存储卡片以供日后使用，例如用于订阅付款或已保存卡片的交易。

<Note>
  令牌类型行为

  令牌的类型(subscription/COF/top-up)在其首次创建时设定并存储在我们的系统中。如果后续的 [Charge Token](/reference/ecom/payments/charge-token) 请求包含不同的令牌类型参数,该参数将被忽略。系统始终使用初始化请求中的原始令牌类型,无论后续扣款请求中提供何种值。
</Note>

## 使用令牌扣款

The [ `/charge-token`](/reference/ecom/payments/charge-token) endpoint is used for recurring payments. It should be used for recurring payment calls and requires a card token to be added to the request. These calls are performed only on the server.

请参见下方代码块中的请求示例：

## 获取令牌数据

`/get-card-token` 端点用于检索已保存卡片令牌的关键数据。它使您能够获取卡片的后四位数字、有效期、品牌和状态等信息。

请求必须在 `BasicInfo` 和 `MachineInfo` 对象中包含以下内容。

```json
{
  "basicInfo": {
    "merchantRequestId": "REQ_GETTOKEN_001"
  },
  "machineInfo": {
    "machineId": "0434334921100366"
  },
  "cardHolderInfo": {
    "cardHolderUniqId": "unique_customer_id_123"
  },
  "tokenInfo": {
    "nayaxTokenId": "NAYAXTOK12345"
  }
}
```

成功的响应将返回令牌数据，如下方示例所示：

```json
{
  "Status": {
    "Verdict": "Approved",
    "Code": 0,
    "StatusMessage": " No Errors "
  },
  "TokenInfo": {
    "4LastDigits": 1234,
    "ExpiryDate": "03/2030",
    "Brand": "Visa",
    "Status": "Active"
  }
}
```

有关请求参数的更多详情，请参阅检索卡片令牌 API 参考页面。

## 删除令牌

删除卡片令牌端点允许您删除已保存的令牌。请求必须在 `BasicInfo` 和 `MachineInfo` 对象中包含以下内容：

```json
{
  "basicInfo": {
    "merchantRequestId": "REQ_DELTOKEN_001"
  },
  "machineInfo": {
    "machineId": "0434334921100366"
  },
  "cardHolderInfo": {
    "cardHolderUniqId": "unique_customer_id_123"
  },
  "tokenInfo": {
    "nayaxTokenId": "NAYAXTOK12345"
  }
}
```

成功的响应将返回删除状态。请参见下方代码块：

```json
{
  "status": {
    "verdict": "Approved",
    "code": 0,
    "statusMessage": "Card token deleted successfully."
  },
  "tokenInfo": null
}
```

有关请求参数的更多详情，请参阅删除令牌 API 参考页面。
