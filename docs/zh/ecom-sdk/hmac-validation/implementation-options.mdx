要集成 HMAC 验证,您可以使用 Nayax 官方 SDK,也可以用您偏好的语言手动实现相关逻辑。

## Nayax SDK（C# 开发者）

C# 开发者可以使用 `SDK.Api.Contracts` 库，该库内置了 HMAC 验证功能。

1. 安装该库：运行 `Install-Package SDK.Api.Contracts`。
2. 使用 HMAC 服务：使用 `HmacService` 类，该类会自动处理所有 HMAC 计算和验证逻辑。

## 自定义实现（非 C# 开发者）

如果您未使用 C#，则应根据以下技术规范复现 Nayax SDK 中的逻辑：

| 规范             | 要求                                                                                           |
| :------------- | :------------------------------------------------------------------------------------------- |
| Hmac 字段名称      | `Hmac` 这是通知正文中的 JSON 字段。                                                                     |
| 签名字符串中的字段      | `NayaxTransactionId`, `MerchantRequestId`, `MachineId`, `RequestTypeName`, `IsApproved`.     |
| 顺序             | 字段必须严格按照上述顺序排列。                                                                              |
| 分隔符            | 冒号（`:`）                                                                                      |
| 缺失字段           | 使用空字符串，但仍必须包含冒号。                                                                             |
| RequestType 格式 | Use the **Enum Name** (e.g., `"Sale"`, `"Auth"`, `"Settlement"`), **not** the integer value. |
| 布尔格式           | `"True"` or `"False"` capital first letter (C# `.ToString()` format).                        |
| 密钥解码           | 64 个十六进制字符解码为 32 个原始字节。                                                                      |
| 哈希输出           | Base64。                                                                                      |
| 排除项            | 哈希计算时不要包含 `RequestDate`、`Hmac` 本身、嵌套对象或数组。                                                   |

## 安全最佳实践

为保持高水平的安全性并防止对您的交易数据进行未经授权的访问，请遵循以下准则：

| 应该做                           | 不应该做            |
| :---------------------------- | :-------------- |
| 存储在环境变量中                      | 硬编码在源代码中        |
| 使用密钥管理服务（如 Azure Key Vault 等） | 提交到版本控制系统       |
| 将 64 个十六进制字符解码为 32 个原始字节      | 直接使用十六进制字符串作为密钥 |
| 处理前始终验证 HMAC                  | 在错误消息中记录密钥      |
