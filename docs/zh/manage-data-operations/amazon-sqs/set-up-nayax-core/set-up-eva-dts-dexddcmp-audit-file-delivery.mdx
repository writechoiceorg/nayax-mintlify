要设置将 DEX 文件传输到 Amazon SQS，您必须拥有访问**Nayax Core 的**相应用户权限。具体来说，需要以下角色：

- VMO经理
- DEX 通知 - SQS

此外，请确保您拥有在队列设置过程中获得的**Amazon SQS 队列 URL** 、**访问密钥**和**密钥**。

<Note>
  先决条件：Dex 读取计划配置

  这些设置说明假设 DEX 读取计划已经配置好。
</Note>

## 操作员级别设置

按照以下步骤在 Nayax Core 中的运营商级别配置 DEX 交付：

1. 前往**管理 > 操作员**。

   <Frame>
     <img src="/images/docs/set-up-eva-dts-dexddcmp-audit-file-delivery/image1.png" />
   </Frame>
2. 找到您想要设置的运营商。
3. 单击**“搜索”**按钮。
4. 选择操作员以显示所有选项卡和详细信息。
5. 转到**Dex**选项卡。
6. 单击打开**Dex Notify**部分。

   <Frame>
     <img src="/images/docs/set-up-eva-dts-dexddcmp-audit-file-delivery/image2.png" />
   </Frame>
7. Set up Dex Transmission to Amazon SQS:
   a. Select the checkbox next to **Amazon SQS** to enable this option.
   b. Enter the SQS credentials (**Access Key**, **Secret Key** and **Queue URL**).
   c. Click the **Validate** button to ensure the Queue credentials are correct.
   d. Enable Encryption (Optional). See the [**Set Up Encryption**](/docs/manage-data-operations/amazon-sqs/set-up-nayax-core/set-up-encryption-on-amazon-sqs-delivery) section for details.

   <Frame>
     <img src="/images/docs/set-up-eva-dts-dexddcmp-audit-file-delivery/image3.png" />
   </Frame>
8. 选择 Dex 格式和过滤器设置：a. 确保格式设置为**“无格式”** 。b. 选择所需的 DEX 读取类型。只有选定的 Dex 来源才会被发送到队列。

   <Frame>
     <img src="/images/docs/set-up-eva-dts-dexddcmp-audit-file-delivery/image4.png" />
   </Frame>
9. 单击**“保存信息”**按钮以应用更改。

   <Frame>
     <img src="/images/docs/set-up-eva-dts-dexddcmp-audit-file-delivery/image5.png" />
   </Frame>

<Warning>
  操作员默认设置与机器级配置

  在操作员级别设置 DEX 交付并不会启动将 DEX 文件交付到队列的操作。相反，这些配置将作为操作员层级下新机器的默认配置。您可以使用**“将 Dex 应用于”**按钮来应用这些设置，也可以按照下文的说明在机器级别执行单独的设置。

  <Frame>
    <img src="/images/docs/set-up-eva-dts-dexddcmp-audit-file-delivery/image6.png" />
  </Frame>
</Warning>

## 机器级别设置

<Note>
  先决条件：Dex 读取计划配置

  这些设置说明假设 DEX 读取计划已经配置好。
</Note>

要在机器级别配置 DEX 传送：

1. 前往**“操作” > ** Nayax 核心中的**机器**。

   <Frame>
     <img src="/images/docs/set-up-eva-dts-dexddcmp-audit-file-delivery/image7.png" />
   </Frame>
2. 找到您想要设置的运营商。
3. 单击**“搜索”**按钮。
4. 选择操作员以显示所有选项卡和详细信息。
5. 转到**Dex**选项卡。
6. 单击打开**Dex Notify**部分。

   <Frame>
     <img src="/images/docs/set-up-eva-dts-dexddcmp-audit-file-delivery/image8.png" />
   </Frame>
7. Set up Dex Transmission to Amazon SQS:
   a. Select the checkbox next to **Amazon SQS** to enable this option.
   b. Enter the SQS credentials (**Access Key**, **Secret Key** and **Queue URL**).
   c. Click the **Validate** button to ensure the Queue credentials are correct.
   d. Enable Encryption (Optional). See the [**Set Up Encryption**](/docs/manage-data-operations/amazon-sqs/set-up-nayax-core/set-up-encryption-on-amazon-sqs-delivery) section for details.

   <Frame>
     <img src="/images/docs/set-up-eva-dts-dexddcmp-audit-file-delivery/image9.png" />
   </Frame>
8. 选择 Dex 格式和过滤器设置：a. 确保格式设置为**“无格式”** 。b. 选择所需的 DEX 读取类型。只有选定的 Dex 来源才会被发送到队列。

   <Frame>
     <img src="/images/docs/set-up-eva-dts-dexddcmp-audit-file-delivery/image10.png" />
   </Frame>
9. 单击**“保存”**按钮以应用更改。

   <Frame>
     <img src="/images/docs/set-up-eva-dts-dexddcmp-audit-file-delivery/image11.png" />
   </Frame>

## JSON 消息字段描述

下表描述了 DEX/DDCMP 审计文件 JSON 消息中包含的字段：

| 字段名称    | 类型       | 描述               |
| ------- | -------- | ---------------- |
| 硬件串行    | `string` | 机器的硬件序列号。        |
| 机器日期    | `string` | 机器记录的日期和时间。      |
| 操作员标识符  | `string` | 机器操作员的唯一标识符。     |
| 起源      | `string` | 数据的来源或起源。        |
| DexData | `string` | 从机器捕获的 DEX 协议数据。 |
