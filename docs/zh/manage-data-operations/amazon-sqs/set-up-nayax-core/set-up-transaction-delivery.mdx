## 先决条件

要设置将交易传输至 Amazon SQS，您必须拥有访问 Nayax Core 的相应用户权限。具体来说，需要以下角色：

- 事务调度器
- 交易报告订阅者

此外，请确保您拥有在队列设置过程中获得的**Amazon SQS 队列 URL** 、**访问密钥**和**密钥**。

## 操作员级别设置

按照以下步骤在 Nayax Core 中配置操作员级别的交易交付：

1. 登录**Nayax Core** 。

   Nayax 核心登录

2. 前往**管理 > 操作员**。

   管理操作员菜单

3. 找到您想要设置的运营商。

4. 单击**“搜索”**按钮。

5. 选择操作员以显示所有选项卡和详细信息。

6. 转到**交易报告**选项卡。

   交易报告选项卡

7. 通过完成以下步骤设置交易分发（Transaction Dispatching）：
   a. 勾选 Amazon SQS 旁边的复选框以启用此选项。
   b. 输入 SQS 凭据（Access Key、Secret Key 和 Queue URL）。
   c. 点击 Validate 按钮以确保队列凭据正确无误。
   d. 启用加密（可选）。有关详细信息，请参见 [Set Up Encryption](/docs/manage-data-operations/amazon-sqs/set-up-nayax-core/set-up-encryption-on-amazon-sqs-delivery) 部分。
   e. 选择要发送的支付方式。仅所选的支付方式会被分发到队列。
   您还可以选择交易类型：

   - 免费交易
   - 已取消的交易
   - 退款交易
   - 拒绝交易

   f. 选择要包含的信息列。有关可用字段的说明，请参阅[“交易 JSON 参考”](/docs/manage-data-operations/amazon-sqs/set-up-nayax-core/transaction-json-reference)页面。

   SQS 配置设置

8. 单击**“保存信息”**按钮以应用更改。

   保存信息按钮

完成这些步骤后，您的交易数据将根据您设置的配置安全地传输到 Amazon SQS。

## 定时文件报告（SFTP/电子邮件）

除了实时 Amazon SQS 交付之外，Nayax Core 还可以定期（每小时或每天）将交易报告发送到 SFTP 服务器或通过电子邮件发送。此设置位于**“交易报告”**选项卡中的**“通过电子邮件/SFTP 定期发送文件报告”**部分。

### 启用并配置报告

1. 在**“交易报告”**选项卡中，滚动到**“通过电子邮件/SFTP 发送的计划文件报告”**部分。
2. 切换**“启用报告”**选项。
3. 配置报告格式：

| 场地    | 描述                                     |
| ----- | -------------------------------------- |
| 文件类型  | 选择**XML**或**CSV 格式**。                  |
| 分隔符   | 请选择制表符、分号、逗号、空格、竖线或其他符号。               |
| 其他    | 自定义分隔符（仅当选择**“其他”**时可用）。 |
| 包含标题行 | 开启此功能可在文件中包含列标题行。                      |

4. 设置**报告间隔**：**每小时**（选择分钟）或**每天**（选择一天中的时间）。

### 连接详情

填写送货连接详情：

| 场地   | 必需的 | 描述           |
| ---- | --- | ------------ |
| IP地址 | 是的  | 服务器主机名或IP地址。 |
| 港口   | 是的  | 服务器端口。       |
| 文件夹  | 不   | 远程目标文件夹路径。   |
| 用户   | 是的  | 用户名。         |
| 密码   | 是的  | 密码。          |
| 指纹   | 不   | 服务器指纹用于验证。   |

点击**“验证”**测试连接，然后保存。
