## 检索机器的最近销售记录

要通过 Machine ID 检索机器的最近销售记录，您需要使用 [Get Last Sales for Machine](/reference/lynx/machines/get-last-sales-for-machine-by-machineid) 端点。通过在请求中提供该机器的 Machine ID，您可以检索到相应机器的唯一标识符。

以下代码块示例展示了该请求，使用 `MachineID` 作为查询参数来筛选结果：

```sh
curl -X 'GET' \
  'https://qa-lynx.nayax.com/operational/v1/machines/<MACHINE_ID>/lastSales' \
  -H 'accept: text/plain' \
  -H 'Authorization: Bearer <YOUR_ACCESS_TOKEN>'
```

<Note>
  Remember to replace `<MACHINE_ID>` with the actual machine's Machine ID.
</Note>

响应将包含有关该机器的详细信息，包括机器的 ID。

### Nayax Core

您也可以在 Nayax Core 中该机器的详情页面找到其最近销售记录，请按照以下步骤操作：

1. 访问 Operations > Machines。
2. 使用搜索功能查找该机器。
3. 从列表中选择该机器。
4. 点击 Info > Last Sales：

## 生成 eReceipt

要生成交易的 eReceipt，您需要使用 [Generate eReceipt](/reference/lynx/ereceipt/generate-ereceipt) 端点。请在请求中提供以下信息：

- TransactionID、MachineAuthorizationTime、SiteID —— 这 3 个字段的值均可通过上文提到的 Last Sales 方法检索获得
- 机器 ID
- 消费者的全名
- 消费者的电子邮件地址

您可以检索到相应机器的唯一标识符。

以下代码块展示了请求示例，使用 `DeviceSerial` 作为查询参数来过滤结果：

```sh
curl -X 'POST' \
  'https://qa-lynx.nayax.com/operational/v1/ereceipt/generate' \
  -H 'accept: text/plain' \
  -H 'Content-Type: application/json-patch+json' \
  -d '{
  "TrasactionID": <TRANSACTION_ID>,
  "TransactionDateTime": <MACHINE_AUTHORIZATION_TIME>,
  "TrasactionSiteID": <SITE_ID>,
  "MachineID": <MACHINE_ID>,
  "FullName": "string",
  "Email": "string"
}'
```

<Note>
  Remember to replace `<TRANSACTION_ID>`, `<MACHINE_AUTHORIZATION_TIME>`, `<SITE_ID>`, `<MACHINE_ID>` with the actual details of the transaction.
</Note>

响应中将包含所发送 eReceipt 的详细信息，包括该 PDF 的链接。

```json
{
  "ReceiptURL": "https://nayax-ereceipt.s3.amazonaws.com/2026/02/12/YaelLawtonStubBilling/OFpQBN-x25GKRbZE8ySlnDnrO44iYVAEUKcw-vKmpoY1/ereceipt4625276436.pdf",
  "EmailSent": true,
  "EreceiptID": 6203654
}
```

1. 访问**报告 > 在线报告 > 动态交易监控器**。

  <img src="/images/docs/getting-a-machines-last-sales-ereceipt-information/image2.png" />

3. 从列表中选择相关交易。
