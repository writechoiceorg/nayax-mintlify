在 Nayax Core 中，机器具有多种属性，例如机器的运营商、机器的型号或 ID 及其状态。您可以通过以下方式访问 Machines 菜单，以查看所有这些属性：

1. 在您的 Dashboard 中，进入 Operations > Machines。
2. 使用搜索功能筛选机器列表。
3. 从列表中选择所需的机器。
4. 打开**“常规信息”**选项卡。

<Frame>
  <img src="/images/docs/retrieve-machine-information/image1.png" />
</Frame>

在此处，您可以查看该机器的所有关键信息，供其他请求使用。

<Warning>
  验证

  请参阅本文档的[“安全和令牌”](/docs/manage-data-operations/lynx-api/security)页面，了解如何访问您的令牌，以及如何正确使用它来验证您的 API 请求。
</Warning>

Lynx API 提供四种以编程方式检索机器信息的方法。请使用与您现有标识符相匹配的部分。

## 通过 `MachineID` 检索机器

使用 [Get Specific Machine Basic Info](/reference/lynx/machines/get-specific-machine-basic-info) 端点检索具有特定 `MachineID` 的机器的详细信息。请在请求的路径参数中插入 `MachineID`。

## 检索所有机器的基本信息

使用 [Get All Machine Basic Info](/reference/lynx/machines/get-all-machine-basic-info) 端点检索所有机器的基本信息，并可通过下列查询参数进行可选筛选。

<EndpointCard title="获取所有机器基本信息" href="/reference/lynx/machines/get-all-machine-basic-info" method="get" />

```sh Request
curl --request GET \
     --url https://qa-lynx.nayax.com/operational/v1/machines \
     --header 'Authorization: Bearer <YOUR_ACCESS_TOKEN>' \
     --header 'accept: application/json'
```

| 范围                  | `Type`   | 描述                        |
| ------------------- | -------- | ------------------------- |
| ResultsLimit        | `int32`  | 要返回的最大结果数；默认设置为 100。      |
| ResultsOffset       | `int32`  | 在开始返回结果之前要跳过的结果数；默认设置为 0。 |
| 演员ID                | `int64`  | 按与机器关联的 actor ID 筛选结果。    |
| 操作员标识符              | `string` | 按运营商标识符筛选结果。              |
| 机器ID                | `int64`  | 按机器 ID 筛选结果。              |
| MachineSerialNumber | `string` | 按机器的序列号筛选结果。              |
| 机器名                 | `string` | 按机器的名称筛选结果。               |
| 设备ID                | `int32`  | 按与机器关联的设备 ID 筛选结果。        |
| VposID              | `int64`  | 按与机器关联的 VPOS ID 筛选结果。     |
