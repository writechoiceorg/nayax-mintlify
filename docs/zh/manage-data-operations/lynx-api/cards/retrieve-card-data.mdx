Lynx API 提供了多个 GET 方法，用于检索有关卡片的详细信息。这些方法可用于查询卡片详情、检查信用额度余额、访问增值历史记录等。

<Warning>
  验证

  请参阅本文档的 [Security & Token](/docs/manage-data-operations/lynx-api/security) 页面，了解如何获取您的令牌（token），以及如何使用它正确地对您的 API 请求进行身份验证。
</Warning>

## 在 Nayax Core 中验证卡片详情

要在 Nayax Core 中查看卡片的详细信息，请进入 Card Management 选项卡，使用搜索栏查找您要检查的卡片，并选择它。

<Frame>
  <img src="/images/docs/retrieve-card-data/image1.png" />
</Frame>

您将看到该卡片的所有详细信息，这与前面提到的 Lynx API 端点检索到的数据相同。

## 获取卡牌

<EndpointCard title="获取卡牌" href="/reference/lynx/cards/get-cards" method="get" />

此通用端点将检索所有卡片的概览信息，包括 Card ID、Card Holder Name 和 Card Status 等基本详细信息。请参见示例查询：

```sh Request
curl --request GET \
     --url https://qa-lynx.nayax.com/operational/v1/cards \
     --header 'accept: application/json' \
     --header 'Authorization: Bearer <YOUR_ACCESS_TOKEN>'
```

通过在请求中添加查询参数，此端点也可以检索特定卡片的详细信息，而不是所有卡片。例如，要获取 Card Unique Identifier 为 `1409315030` 的卡片的详细信息，您可以使用下方代码块中的查询。

```sh Request
curl --request GET \
     --url 'https://qa-lynx.nayax.com/operational/v1/cards?CardUniqueIdentifier=1409315030%20' \
     --header 'accept: application/json' \
     --header 'Authorization: Bearer <YOUR_ACCESS_TOKEN>'
```

卡片的以下任何详细信息都可以用作查询参数：

- `CardID`
- `CardUniqueIdentifier`
- `CardDisplayNumber`
- `CardMobileNumber`
- `ExternalApplicationUserID`
- `CardEmail`
- `CardHolderName`

## 通过卡片显示编号获取卡片

<EndpointCard title="通过卡片显示编号获取卡片" href="/reference/lynx/cards/get-card-by-the-cards-display-number" method="get" />

它根据卡片的显示号码（即卡片的物理卡号）获取卡片详细信息。这非常适用于只能获取显示号码作为参考的情况。

请参见下方代码块中的查询示例：

```sh
curl --request GET \
     --url https://qa-lynx.nayax.com/operational/v1/cards/displayNumber/{CardDisplayNumber} \
     --header 'Authorization: Bearer <YOUR_ACCESS_TOKEN>' \
     --header 'accept: application/json'
```

<Note>
  将 `{CardDisplayNumber}` 替换为该卡片的实际号码。
</Note>

## 获取预付卡

<EndpointCard title="获取预付卡" href="/reference/lynx/cards/get-prepaid-card" method="get" />

它检索与特定 CardID 关联的预付卡的详细信息。可用于管理预付卡或验证其状态和属性。

请参见下方代码块中的查询示例：

```sh Request
curl --request GET \
     --url https://qa-lynx.nayax.com/operational/v1/cards/CardID/prepaid \
     --header 'accept: application/json' \
     --header 'Authorization: Bearer <YOUR_ACCESS_TOKEN>'

```

<Note>
  将 `{CardID}` 替换为该卡片的实际 `CardID`。
</Note>

响应中将包含所请求卡片的详细信息。

## 从卡中获得信用

<EndpointCard title="从卡中获得信用" href="/reference/lynx/cards/get-the-credit-from-a-card" method="get" />

此选项通过唯一标识符获取特定卡片的当前信用额度余额。可用于监控卡片余额，例如预付卡或技术员卡的余额。

请参见下方代码块中的查询示例：

<CodeGroup>
  ```sh Request
  curl --request GET \
       --url https://qa-lynx.nayax.com/operational/v1/cards/{CardUniqueIdentifier}/credit \
       --header 'Authorization: Bearer <YOUR_ACCESS_TOKEN>' \
       --header 'accept: application/json'
  ```

  ```json Response
  {
    "value": 5320
  }
  ```
</CodeGroup>

<Note>
  将 `{CardUniqueIdentifier}` 替换为该卡的实际标识符。
</Note>

响应中将包含该卡片的信用额度余额。

## 获取卡的重新估值

<EndpointCard title="获取卡的重新估值" href="/reference/lynx/cards/get-cards-revalue" method="get" />

通过卡片的唯一标识符检索与特定卡片关联的增值金额。请参见下方的查询示例：

```sh Request
curl --request GET \
     --url https://qa-lynx.nayax.com/operational/v1/cards/{CardUniqueIdentifier}/revalue \
     --header 'Authorization: Bearer <YOUR_ACCESS_TOKEN>' \
     --header 'accept: application/json'
```

<Note>
  将 `{CardUniqueIdentifier}` 替换为该卡的实际标识符。
</Note>

响应中将包含该卡片的增值金额。
