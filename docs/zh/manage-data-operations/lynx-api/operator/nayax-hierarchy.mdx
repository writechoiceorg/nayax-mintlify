Nayax 使用结构化系统来组织和管理实体，以反映现实世界的业务运营。此层次结构是管理系统内操作员、机器和其他实体的基础。这些实体包括：

- **分销商**：层次结构中最顶层的实体，代表管理多个运营商的区域或国家级组织。分销商负责监督运营，但不能拥有父实体，也不能通过 Lynx API 创建。
- **运营商**：直属分销商的企业或组织。运营商负责管理其子层级（区域、路线、机器），并处理库存、机器和用户权限。运营商之间不能共存于其他运营商之下。
- **机器**：指包含库存和销售数据的实体自动售货机。机器根据运营商的层级结构分配到特定的路线或区域，是跟踪交易和发出警报的重要组成部分。

请参阅下一节中包含这些实体的层次结构的示例。

## 层次结构示例

下图说明了 Nayax 内的层次结构，显示了实体如何从顶层（分销商）到最低层（机器）进行组织。

<Frame>![](/images/docs/nayax-hierarchy/image1.png)</Frame>

在哪里：

- Distributor（分销商）：监管整个结构的最高级别实体。在此示例中，它管理两个运营商：Operator A 和 Operator B。
  - Operator A 管理 Area A 和 Area B。每个区域都包含用于对机器进行分组以便于维护的路线。
    - 区域 A 包括机器 1 和 2 的路线 A 以及机器 3 和 4 的路线 B。
    - 区域B包括路线C，管理机器5和机器6。
  - **操作员 B**管理区域 C，其中包含路线 D，以及机器 7 和 8。

如您所见，机器与特定路线绑定，路线按区域分组。操作员独立组织其层级，但仍受分销商的监督。

<Note>
  Lynx API 参与者

  在 Nayax Core 中，运营商及其类型（例如 "Area" 和 "Routes"）只是用于区分它们的命名方式。而在 Lynx API 中，它们统称为 Actor，仅通过 `ActorTypeID` 加以区分。
</Note>

## 机构

在Nayax系统中，各机构是与运营商并行的独特实体，直接隶属于分销商。它们促进多个运营商在特定地点或特定用例上的共享运营，例如在多个运营商的机器上实施统一的预付卡系统。

参见下图：

<Frame>![](/images/docs/nayax-hierarchy/image2.png)</Frame>

在哪里：

- 处于层级结构的顶端，经销商负责监管机构和运营商。在本例中，经销商负责管理运营商 A 和一个机构。
- 机构直接与分销商连接，不属于运营商的层级结构。它们与地点相关联，例如地点 1（绿色）和地点 2（红色），这些地点可能包含由一个或多个运营商提供服务的机器。

机构允许多个运营商的机器共享同一套预付卡或支付系统。例如，位于地点 1 的机器和另一个运营商的机器可以使用相同的员工卡系统。这种结构使得机构对于多个运营商之间的协同运营非常有价值，从而实现了跨地点的灵活性和集成。
