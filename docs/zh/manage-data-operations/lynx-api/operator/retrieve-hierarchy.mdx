Nayax Core 会在左侧边栏的 Operator 菜单中显示运营商的层级结构。如果您查看其数据，Company 部分将详细说明其运营商 ID 和 Parent Operator（父运营商）。

## Lynx API

您还可以使用 Lynx API 通过使用以下端点以编程方式检索操作员的层次结构及其详细信息：

<EndpointCard title="获取当前用户的 Actor 层次结构" href="/reference/lynx/actors/get-actor-hierarchy-for-current-user" method="get" />

### 获取运算符层次结构

[Get Actor Hierarchy for Current User](/reference/lynx/actors/get-actor-hierarchy-for-current-user) 端点通过在请求的查询参数中提供 `ActorID`，检索当前用户或指定 Actor/运营商及其子实体（如区域、路线和机器）的层级结构。请参阅下方代码块中的示例请求：

```sh
curl --request GET \
     --url 'https://qa-lynx.nayax.com/operational/v1/actors/hierarchy?ActorID=<ID>' \
     --header 'accept: application/json'
```

可用的查询参数有：

| 范围                    | 描述                                     |
| :-------------------- | :------------------------------------- |
| `ActorID`             | 您想要检索其层次结构的操作员或参与者的唯一 ID。              |
| `StatusID`            | 筛选层次结构以仅包含具有特定状态的参与者。使用此选项仅检索活动或非活动实体。 |
| `HierarchyLevelLimit` | 指定要返回的层次结构的最大深度。例如，1 表示直系子代，2 表示孙代。    |

<Note>
  Lynx API 参与者

  在 Nayax Core 中，运营商及其类型（例如 "Area" 和 "Routes"）只是用于区分它们的命名方式。而在 Lynx API 中，它们统称为 Actor，仅通过 `ActorTypeID` 加以区分。
</Note>

### 回复

请求成功后，端点将返回一个 Operator 对象数组。请参阅以下代码块中的示例：

```json
{
  "ParentActorID": 0,
  "ActorID": 12345,
  "ActorDescription": "Operator A",
  "ActorTypeID": 2,
  "ActorStatus": 1,
  "ActorHierarchyLevel": 0,
  "ActorChildren": [
    {
      "ParentActorID": 12345,
      "ActorID": 23456,
      "ActorDescription": "Area A",
      "ActorTypeID": 3,
      "ActorStatus": 1,
      "ActorHierarchyLevel": 1,
      "ActorChildren": [
        {
          "ParentActorID": 23456,
          "ActorID": 34567,
          "ActorDescription": "Route A",
          "ActorTypeID": 4,
          "ActorStatus": 1,
          "ActorHierarchyLevel": 2,
          "ActorChildren": [
            {
              "ParentActorID": 34567,
              "ActorID": 45678,
              "ActorDescription": "Machine 1",
              "ActorTypeID": 5,
              "ActorStatus": 1,
              "ActorHierarchyLevel": 3,
              "ActorChildren": []
            }
          ]
        }
      ]
    }
  ]
}
```

As you can see above, this is the hierarchy for the Operator with the `ActorID: 12345` showing each of its children operators. The table below describes each of the response's body parameters:

| 范围                    | 描述                    |
| :-------------------- | :-------------------- |
| `ParentActorID`       | 指定演员的父演员的 ID。         |
| `ActorID`             | 当前演员的唯一 ID。           |
| `ActorDescription`    | 演员的描述或姓名。             |
| `ActorTypeID`         | 参与者的类型（例如，操作员、区域、路线）。 |
| `ActorStatus`         | 参与者的状态（例如，活跃、不活跃）。    |
| `ActorHierarchyLevel` | 等级制度中演员的级别。           |
| `ActorChildren`       | 当前演员的子演员 ID 或详细信息列表。  |
