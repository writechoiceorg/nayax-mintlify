在 Nayax Core 中，可以对运营商执行多种操作，例如更新或检索运营商的信息，所有这些操作均在 Administration > Operators 菜单中执行。

<Frame>
  <img src="/images/docs/operator-actions/image1.png" />
</Frame>

这些操作也可以通过 Lynx API 以编程方式执行，使用以下端点：

<Columns cols={2}>
  <EndpointCard title="通过 ActorID 获取演员详情" href="/reference/lynx/actors/get-actor-details-by-actorid" method="get" />

  <EndpointCard title="更新操作符/子操作符" href="/reference/lynx/actors/update-operatorsub-operator" method="put" />
</Columns>

<Warning>
  验证

  请参阅本文档的[“安全和令牌”](/docs/manage-data-operations/lynx-api/security)页面，了解如何访问您的令牌，以及如何正确使用它来验证您的 API 请求。
</Warning>

## 检索操作员详细信息

[通过 ActorID 获取参与者详情](/reference/lynx/actors/get-actor-details-by-actorid)可检索特定参与者的综合信息。此端点提供通过提供的 ActorID 标识的参与者的详细属性，包括个人资料信息和相关配置。

<CodeGroup>
  ```sh Request
  curl --request GET \
       --url https://qa-lynx.nayax.com/operational/v1/actors/{ActorID} \
       --header 'Authorization: Bearer <YOUR_ACCESS_TOKEN>' \
       --header 'accept: application/json'
  ```

## 更新运算符

[Update Operator/Sub-Operator](/reference/lynx/actors/update-operatorsub-operator) 端点允许您编辑指定运营商的信息。

<CodeGroup>
  ```sh Request
  curl --request PUT \
       --url https://qa-lynx.nayax.com/operational/v1/actors/{ActorID} \
       --header 'Authorization: Bearer <YOUR_ACCESS_TOKEN>' \
       --header 'accept: application/json' \
       --header 'Content-Type: application/json' \
       --data '{
         "ActorDescription": "Updated Main Branch",
         "ActorCode": 1002,
         "ActorAddress": "456 New Street, Metropolis",
         "ActorContact": "updated-contact@branch.com",
         "ActorTypeID": 16,
         "ActorStatus": 1,
         "CurrencyID": 840,
         "CountryID": 840,
         "GeoCity": "Metropolis",
         "GeoLongitude": -77.0364,
         "GeoLatitude": 38.8951
       }'
  ```
