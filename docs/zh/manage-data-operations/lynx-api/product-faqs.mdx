本常见问题页面解答了在 Lynx API 中管理产品地图和更新相关的常见问题和疑难点。

## 为什么仅更新一个产品会使我的其他产品变为空值？

当您为单个产品发送更新请求时，系统会重写整个产品地图。这意味着，如果您在请求中未包含所有产品及其当前值，被遗漏的产品将被设置为 "null" 或默认值。为避免这种情况，您应始终发送完整的产品列表，并仅对您想要更新的特定产品进行更改。

## 是否已添加发布产品图片的选项？

尚未添加发布产品图片的选项。不过，该功能仍在开发路线图中，未来版本中可能会推出。

## 是否会提供用于处理产品地图模板的选项（创建、复制、更新到机器）？

是的，处理产品地图模板的选项已列入路线图。不过，这些功能近期内尚不会推出。

## 如何通过 Lynx 重置/删除机器的产品地图？

`MachineProductID` 用作产品地图中每一行的标识符，您可以以此为参照对各行进行修改。但从技术上讲，您无法“删除”或“重置”整个产品地图。您可以手动将每一行的值更改为所需的值，但这些行仍将保留在地图中。
