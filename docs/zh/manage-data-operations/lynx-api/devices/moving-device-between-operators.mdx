Lynx API 允许您在 Nayax 系统的层级结构中，将设备从一个运营商重新分配给另一个运营商。这对于管理多个运营商、地区或组织单位的企业尤其有用，可确保在设备转手时对其进行无缝控制和跟踪。

本页面介绍如何使用 Lynx API 将设备从一个运营商转移到另一个运营商。在下面的分步指南中，您将使用以下端点：

<EndpointCard title="Move Devices" href="/reference/lynx/devices/move-devices-to-actor-id" method="put" />

<Warning>
  验证

  请参阅本文档的[“安全和令牌”](/docs/manage-data-operations/lynx-api/security)页面，了解如何访问您的令牌，以及如何正确使用它来验证您的 API 请求。
</Warning>

## 转移设备

使用 [Move Devices](/reference/lynx/devices/move-devices-to-actor-id) 端点将设备分配或转移到指定的 `actorId`。请求正文应包含设备标识符（例如序列号）的数组。请参见下方代码块中的示例请求：

```sh Request
curl -X PUT "https://qa-lynx.nayax.com/operational/v1/devices/move/<ACTOR_ID>" \
-H "Authorization: Bearer <YOUR_ACCESS_TOKEN>" \
-H "Content-Type: application/json" \
-d '[
  "SN12345X",
  "SN67890Y"
]'
```

<Note>
  路径参数

  Replace `<ACTOR_ID>` with the target actor's unique identifier for the new operator.
</Note>

如您所见，上述请求的正文包含两个将被转移到目标运营商的设备序列号。响应将返回每台被转移设备的状态。请参见下方代码块：

```json Response
[
  {
    "HW_serial": "SN12345X",
    "actor_id": 3001492652,
    "is_connected": true
  },
  {
    "HW_serial": "SN67890Y",
    "actor_id": 3001492652,
    "is_connected": false
  }
]
```

在哪里：

| 范围             | 描述                     |
| :------------- | :--------------------- |
| `HW_serial`    | 该设备的硬件序列号。             |
| `actor_id`     | 确认该设备已被转移到的新 `actorId` |
| `is_connected` | 指示该设备在转移之前是否已连接到虚拟机。   |
