运营商产品是您目录中的单个商品，例如“Energy Drink”或“Potato Chips”，每个商品都有其自己的定价、条形码和分组归属。

在 Nayax Core 中，您可以通过以下方式创建产品：

1. 转到 Administration > Products。
2. 点击 Create > Add Product。
3. 填写表格中的必填字段：
   - Operator：选择该产品所属的运营商。
   - Group：选择该运营商下此产品所属的产品组。
   - Product Name：产品的名称。
   - Status：产品是否处于激活状态。
4. 点击“保存”以完成。

## 通过 API 创建产品

使用 [Create New Operator Product](/reference/lynx/products/create-new-operator-product) 端点以编程方式创建产品。

<EndpointCard title="创建新的运营商产品" href="/reference/lynx/products/create-new-operator-product" method="post" />

<Warning>
  验证

  请参阅本文档的[“安全和令牌”](/docs/manage-data-operations/lynx-api/security)页面，了解如何访问您的令牌以及如何正确使用它们来验证您的 API 请求。
</Warning>

<br />

发送 POST 请求，将 `OperatorID` 作为路径参数，并在请求正文中提供产品详细信息：

```sh Request
curl --request POST \
     --url https://qa-lynx.nayax.com/operational/v1/operators/{OperatorID}/products \
     --header 'Authorization: Bearer <YOUR_ACCESS_TOKEN>' \
     --header 'accept: application/json' \
     --header 'Content-Type: application/json' \
     --data '{
       "NayaxProductID": 12345,
       "ProductGroupID": 1001,
       "ActorID": 67890,
       "ProductManufacturerID": 55,
       "ProductName": "Energy Drink",
       "ProductCatalogNumber": "ED123",
       "ProductBarcode": "0123456789123",
       "ProductPackageQuantity": 24,
       "ProductDescription": "A refreshing energy drink.",
       "ProductVolumeTypeID": 3,
       "DEXProductName": "Energy_Drink",
       "ProductCostPrice": 1.20,
       "ProductDefaultRetailPrice": 2.50,
       "ProductMinimumPickQTY": 5,
       "ProductStatus": 1,
       "ProductCashPrice": 2.40,
       "ProductCreditCardPrice": 2.60,
       "ProductPrepaidCardPrice": 2.50,
       "ProductExternalPrepaidCardPrice": 2.55,
       "ProductMemberTypePriceBit": true,
       "ProductPictureURL": "https://example.com/energy-drink.jpg",
       "CaloriesPer100g": 45.0,

```

<Note>
  路径参数

  请确保将 `OperatorID` 作为路径参数发送，以指定该产品所属的运营商。
</Note>

<br />

### 一次创建多个产品

您可以使用[“创建机器产品”](/reference/lynx/machine-products/create-machine-products)端点为单台机器创建多个产品。这样，您可以在一个请求中添加多个产品，即使它们属于不同的产品组。

```sh Request
curl -X POST "https://qa-lynx.nayax.com/operational/v1/machines/<MACHINE_ID>/machineProducts" \
-H "Authorization: Bearer <YOUR_ACCESS_TOKEN>" \
-H "Content-Type: application/json" \
-d '[
  {
    "MachineID": 5001,
    "ProductName": "Cola Drink",
    "MDBCode": 1,
    "CashPrice": 1.75,
    "CreditCardPrice": 1.80,
    "RetailPrice": 1.85,
    "ProductGroupID": 201,
    "DEXProductName": "Cola Drink",
    "IsActive": true
  },
  {
    "MachineID": 5001,
    "ProductName": "Potato Chips",
    "MDBCode": 2,
    "CashPrice": 2.25,
    "CreditCardPrice": 2.30,
    "RetailPrice": 2.40,
    "ProductGroupID": 202,
    "DEXProductName": "Potato Chips",
    "IsActive": true
  }
]'
```

<Note>
  路径参数

  在上面的例子中，替换`<MACHINE_ID>`您需要提供实际的机器标识符来添加产品。
</Note>

### 更新产品

您可以使用 [Update Specific Product](/reference/lynx/products/update-specific-product) 端点，通过在路径参数中添加产品的 `NayaxProductID`，并在正文请求中提供要更改的信息，来更改产品的详细信息。如下例请求所示：

```sh Request
curl --request PUT \
     --url https://qa-lynx.nayax.com/operational/v1/products/{NayaxProductID} \
     --header 'Authorization: Bearer <YOUR_ACCESS_TOKEN>' \
     --header 'accept: application/json' \
     --header 'Content-Type: application/json' \
     --data '{
       "ProductName": "Updated Chocolate Bar",
       "ProductGroupID": 1002,
       "ProductDefaultRetailPrice": 3.00,
       "ProductCashPrice": 2.90,
       "ProductCreditCardPrice": 3.10,
       "ProductDescription": "Updated description for the chocolate bar.",
       "IsActive": true
     }'
```

主体参数与创建产品请求中的相同。如果请求成功，响应将包含更新后的产品详细信息。

After creating your products, you're ready to [map them to machines](/docs/manage-data-operations/lynx-api/inventory-management/machine-products-1).
