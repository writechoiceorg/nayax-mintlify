退款流程始于提交退款申请，申请中包含退款金额和原因等详细信息。这将通知客户退款正在处理中。我们将在本页面更详细地描述此流程。

<Warning>
  仅限信用卡

  只有通过信用卡进行的交易才能申请退款。预付卡交易不适用此选项。
</Warning>

在 Nayax Core 中，您可以通过**动态交易监控器**申请退款。请按照以下步骤操作。

1. 访问**报告 > 在线报告 > 动态交易监控器**。
2. 搜索要退款的交易。您可以使用筛选功能来简化操作。
3. 右键单击您想要退款的交易。
4. 点击**申请退款**

   <Frame>
     <img src="/images/docs/request-refunds/image1.png" />
   </Frame>
5. 将会弹出一个窗口。在其中，选择全部退款还是部分退款，并填写退款原因。
6. 点击**申请退款**。

   <Frame>
     <img src="/images/docs/request-refunds/image2.png" />
   </Frame>

### 发起请求

要启动请求，请使用[“请求退款”](/reference/lynx/payment/request-a-payment-refund)端点申请退款。在请求的正文参数中提供与退款相关的属性，如下所示：

```sh Request
curl --request POST \
     --url https://qa-lynx.nayax.com/operational/v1/payment/refund-request \
     --header 'accept: application/json' \
     --header 'content-type: application/json' \
     --data '
{
  "RefundAmount": 0,
  "RefundEmailList": "string",
  "RefundReason": "string",
  "TransactionId": 0,
  "SiteId": 0,
  "MachineAuTime": "2024-10-10T16:30:37.179Z"
}
'
```

下面，您可以看到每个参数的描述：

| 姓名              | 类型              | 描述                     |
| :-------------- | :-------------- | :--------------------- |
| 退款金额            | `double / null` | 退还给客户的金额。              |
| RefundEmailList | `string / null` | 用于通知退款的电子邮件地址列表，以逗号分隔。 |
| 退款原因            | `string / null` | 要求退款的原因。               |
| 交易ID            | `int64`         | 请求退款的交易的唯一标识符。         |
| 站点编号            | `int32`         | 发生交易的站点的唯一标识符。         |
| 机器AuTime        | `date-time`     | 机器的授权时间与交易相关联。         |

### 如何获取TransactionID和SiteID

Retrieve `TransactionId` and `SiteId` from the Lynx API using the [Get Last Sales for Machine](/reference/lynx/machines/get-last-sales-for-machine-by-machineid) endpoint with the `MachineID`. The response object includes both values.

## 请求状态

一旦您发送请求，通知电子邮件将会发送到请求参数中输入的电子邮件地址。

该退款现处于 `pending`（待处理）状态，需要被批准或拒绝。详情请参阅 [Approve or Decline a Refund](/docs/manage-data-operations/lynx-api/refunds/approve-or-decline-a-refund)。
