退款申请成功后，交易将保持**待处理**状态，直至获得批准或被拒绝。您可以在 Nayax 核心页面的**“报告”**中查看此状态。 ** > 在线报告 > 动态事务监控**。请遵循以下步骤：

1. 使用搜索功能查找交易。然后，选择**“已申请退款”**并按**“状态”**进行筛选。
2. 右键单击列表中的交易。
3. 单击**“批准/拒绝退款”** 。

   <Frame>
     <img src="/images/docs/approve-or-decline-a-refund/image1.png" />
   </Frame>
4. 将出现一个弹出窗口。点击**“拒绝退款”**或**“批准退款”** 。

   <Frame>
     <img src="/images/docs/approve-or-decline-a-refund/image2.png" />
   </Frame>

### 批准请求

使用“批准付款退款”端点批准退款。以下代码块显示了请求对象及其正文参数：

```sh Request
curl --request POST \
     --url https://qa-lynx.nayax.com/operational/v1/payment/refund-approve \
     --header 'accept: application/json' \
     --header 'content-type: application/json' \
     --data '
{
  "IsRefundedExternally": true,
  "RefundDocumentUrl": "string",
  "TransactionId": 0,
  "SiteId": 0,
  "MachineAuTime": "2024-10-10T16:30:37.179Z"
}
'
```

### 拒绝请求

要拒绝退款，请使用“拒绝付款退款”端点。以下代码块显示了请求对象及其正文参数。

```sh Request
curl --request POST \
     --url https://qa-lynx.nayax.com/operational/v1/payment/refund-decline \
     --header 'accept: application/json' \
     --header 'content-type: application/json' \
     --data '
{
  "DeclineReason": "string",
  "TransactionId": 0,
  "SiteId": 0,
  "MachineAuTime": "2024-10-10T16:30:37.179Z"
}
'
```

有关每个参数的更多详细信息，请参阅下表：

| 姓名       | 类型          | 描述             |
| -------- | ----------- | -------------- |
| 拒绝原因     | `string`    | 拒绝退款请求的原因。     |
| 交易ID     | `int64`     | 拒绝退款的交易的唯一标识符。 |
| 站点编号     | `int32`     | 发生交易的站点的唯一标识符。 |
| 机器AuTime | `date-time` | 机器的授权时间与交易相关联。 |

<Warning>
  `TransactionId`、`SiteId` 和 `MachineAuTime` 必须与退款请求中发送的值保持一致。
</Warning>

## 请求状态

退款申请获批或被拒后，消费者将收到一封发送至退款申请中提供的邮箱地址的电子邮件。申请状态也会更新。您可以在 Nayax Core 的**“报告”**页面查看此信息。 ** > 在线报告 > 动态交易监控器**。
