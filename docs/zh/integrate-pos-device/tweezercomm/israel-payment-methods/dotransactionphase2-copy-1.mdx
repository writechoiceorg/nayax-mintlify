`removeSubRetailer` 方法用于从终端移除一个子零售商。

## 要求

<Note>
  JSON-RPC 2.0

  TweezerComm 中的所有方法都遵循 JSON-RPC 2.0 结构。有关请求的结构和发送方式的更多详细信息，请参阅[“集成”部分](/docs/integrate-pos-device/tweezercomm/tweezercomm-get-started)。
</Note>

请在请求中确保配置以下内容：

- `method` 字段必须设置为 `removeSubRetailer`。
- The service must be set to `ashrait` in the `params`.

```json
{
    "jsonrpc": "2.0",
    "method": "removeSubRetailer",
    "params": [
        "ashrait",
        {
            "retailerId": "0883012"
        }
    ],
    "id": "123454352"
}
```

### 请求参数

`params` 数组包含服务名称，其后跟一个字典。

| 姓名           | 类型     | 描述            |
| :----------- | :----- | :------------ |
| `retailerId` | string | 要移除的子零售商的 ID。 |

## 回复

成功调用会返回状态代码和消息。

```json
{
    "jsonrpc": "2.0",
    "id": "123454352",
    "result": {
        "statusCode": 0,
        "statusMessage": "OK"
    }
}
```

### 响应参数

结果对象包含以下字段：

| 姓名                     | 类型     | 描述                 |
| :--------------------- | :----- | :----------------- |
| `statusCode`           | 整数     | 操作的状态。`0` 表示成功。    |
| `statusMessage`        | string | 描述状态的消息，例如 `"OK"`。 |
