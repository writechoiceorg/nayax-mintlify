`getTransactionByUid` 方法可通过唯一标识符 (UID) 返回交易。如果未提供 UID，该方法将返回最后一笔交易。

## 要求

<Note>
  JSON-RPC 2.0

  TweezerComm 中的所有方法都遵循 JSON-RPC 2.0 结构。有关请求的结构和发送方式的更多详细信息，请参阅[“入门指南”](/docs/integrate-pos-device/tweezercomm/tweezercomm-get-started) 。
</Note>

请在请求中确保配置以下内容：

- `method` 字段必须设置为 `getTransactionByUid`。
- The service must be set to `ashrait` in the `params`.

```json
{
    "jsonrpc":"2.0",
    "method":"getTransactionByUid",
    "params":[
        "ashrait",
        {
           "originalUID":"24071711405408830122531"
        }
    ],
    "id":"123454352"
}
```

### 请求参数

<br />

下表描述了此方法的请求参数。

| 姓名            | 类型     | 描述                             |
| :------------ | :----- | :----------------------------- |
| `originalUID` | string | 可选。要检索的交易的 UID。如果省略，将返回最近一笔交易。 |
