`addSubRetailer` 方法用于将子零售商加载到终端中。此方法仅在 Shva 处将主零售商定义为 RAV-SAPAK 时才可使用。

<Warning>
  使用 `doPeriodic`

  加载新零售商后，您应使用 `doPeriodic` 方法。
</Warning>

## 要求

<Note>
  JSON-RPC 2.0

  TweezerComm 中的所有方法都遵循 JSON-RPC 2.0 结构。有关请求的结构和发送方式的更多详细信息，请参阅[“集成”部分](/docs/integrate-pos-device/tweezercomm/tweezercomm-get-started)。
</Note>

请在请求中确保配置以下内容：

- `method` 字段必须设置为 `addSubRetailer`。
- The service must be set to `ashrait` in the `params`.

请求需要包含 `retailerId` 列表。

```json
{
    "jsonrpc": "2.0",
    "method": "addSubRetailer",
    "id": "123454352",
    "params": [
        "ashrait",
        {
            "retailerId": [
                "0883806",
                "0884806"
            ]
        }
    ]
}
```

### 请求参数

下表描述了此方法的请求参数。

| 姓名           | 类型    | 描述     |
| :----------- | :---- | :----- |
| `retailerId` | array | 零售商列表。 |

## 回复

成功的请求将返回以下响应：

```json
{
    "jsonrpc": "2.0",
    "id": "123454352",
    "result": {
        "statusCode": 0,
        "statusMessage": "OK"
    }
}
```
