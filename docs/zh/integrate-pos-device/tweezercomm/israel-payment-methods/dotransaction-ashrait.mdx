`doTransaction` 方法可在 TweezerComm 协议中发起并处理支付交易。它使销售点 (POS) 系统或电子收银机 (ECR) 等客户端应用程序能够与 Agamento 支付终端交互，以安全地执行交易。

此方法支持多种交易类型，包括：

- 常规交易：使用信用卡或闭环支付方式的标准支付。
- 退款：对先前交易进行部分或全额退款。
- 预授权：在客户卡上预留指定金额，以供后续确认。
- 延迟付款：即使在没有即时网络连接的情况下也能批准交易。
- MOTO（邮购/电话订购）交易：安全处理无卡交易。
- 预授权完成与取消：完成或撤销先前已授权的交易。

以下各部分将详细介绍请求参数：

## 要求

<Note>
  JSON-RPC 2.0

  TweezerComm 中的所有方法都遵循 JSON-RPC 2.0 结构。有关请求的结构和发送方式的更多详细信息，请参阅[“入门指南”](/docs/integrate-pos-device/tweezercomm/tweezercomm-get-started) 。
</Note>

请在请求中确保配置以下内容：

- `method` 字段需要设置为 `doTransaction`。
- The service must be set to `ashrait` in the `params`.

```json
{
    "jsonrpc": "2.0",
    "method": "doTransaction",
    "params": [
        "ashrait",
        {
            "amount": 2000,
            "vuid": "1234567890",
            "currency": "376",
            "creditTerms": 1,
            "tranCode": 1,
            "tranType": 1,
            "cardNumber": "4580000000000000",
            "expDate": "2308",
            "cvv": "666"
        }
    ],
    "id": 1
}
```
