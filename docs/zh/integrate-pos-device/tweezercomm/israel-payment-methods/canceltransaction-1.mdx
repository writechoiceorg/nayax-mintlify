`cancelTransaction` 方法用于取消已批准并已在数据库中注册的交易。

<Warning>
  只要该交易仍存储在 AEA 数据库中，就可以取消该交易；该数据库会在调用 `doPeriodic` 进行日终批处理后被清空。
</Warning>

客户端也可以通过使用适当的 tranCode 调用 `doTransactionPhase1` 来实现此操作。

## 要求

<Note>
  JSON-RPC 2.0

  TweezerComm 中的所有方法都遵循 JSON-RPC 2.0 结构。有关请求的结构和发送方式的更多详细信息，请参阅[“集成”部分](/docs/integrate-pos-device/tweezercomm/tweezercomm-get-started)。
</Note>

请在请求中确保配置以下内容：

- `method` 字段必须设置为 `cancelTransaction`。
- 该服务必须在参数（params）中设置为 `ashrait`。

请求包含一个参数列表，第一个参数为服务名称，目前固定为 `ashrait`，第二个参数为包含所有其他交易特定详情的字典。

```json
{
    "jsonrpc": "2.0",
    "method": "cancelTransaction",
    "params": [
        "ashrait",
        {
            "originalUID": "24071711552508830122539",
            "vuid": "1234567"
        }
    ],
    "id": "123454352"
}
```

### 请求参数

下表描述了此方法的请求参数。

| 姓名            | 类型     | 描述              |
| :------------ | :----- | :-------------- |
| `originalUID` | string | 要取消的交易的原始 UID。  |
| `vuid`        | string | 供应商（客户）唯一交易标识符。 |
