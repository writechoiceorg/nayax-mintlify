# 以色列

在以色列，您的设备将配置为测试终端：

这样做是为了能够在不激活设备的情况下访问该应用程序。

## 设置测试终端配置

您需要连续点击 Agamento 的徽标 7 次——您会看到"Login"（登录）页面从"Login to DCS"切换为"Login to DCS QA2"，同时出现一个"Skip"（跳过）按钮：请点击该"Skip"按钮。

然后再次进入技术人员菜单，这次进入"Payment Services"（支付服务）：

<Note>
  使用测试终端配置时（仅适用于以色列），客户必须确保使用测试卡：这与 Nayax 无关，因为我们不发行卡片，测试卡可通过多家发卡机构获取，例如 Max、Cal、Isracard 等。如果在配置为测试终端的情况下出示真实信用卡，该交易将被拒绝，因为该卡会被定向到 SHVA 的测试服务器（该服务器仅接受测试卡）
</Note>

## 常规设置

您还需要进入技术人员菜单 -> 设置，确保设备具有正确的 POS 类型（集成类型）。

我们将移除"Amount dialog"（金额对话框）——这是在消费者出示卡片之前弹出的一个屏幕，询问消费者是否同意该笔交易（通常客户希望消费者在发起交易后立即出示卡片）。

# 全球

在以色列以外的任何国家，您将使用一个"存根"（虚拟）计费提供商来代替测试终端，它会模拟真实计费提供商批准您所出示卡片的过程——但实际上不会与真实计费提供商产生任何联系，您也不会因此产生任何交易费用。

在这种情况下，您可以放心使用真实信用卡，无需担心产生费用。
