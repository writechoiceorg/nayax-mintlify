`startPinpadTransaction` 方法在支付终端上启动 PinPad 流程，指示终端准备好通过磁条读卡（MSR）、非接触式（Tap）或接触式方式接收卡片信息。如果是接触式交易，交易将继续进入 doPinPadTransaction，否则不会——详见 [Pinpad Mode](/docs/integrate-pos-device/tweezercomm/pinpad-mode) 相关内容。响应将通过 `panEntryMode` 参数提供所使用的支付方式。

此方法从物理设备采集原始卡片数据（Track 2 或 EMV 标签），为后续的授权步骤做好准备。

## 要求

<Note>
  JSON-RPC 2.0

  TweezerComm 中的所有方法都遵循 JSON-RPC 2.0 结构。有关请求的结构和发送方式的更多详细信息，请参阅[“入门指南”](/docs/integrate-pos-device/tweezercomm/tweezercomm-get-started) 。
</Note>

请在请求中确保配置以下内容：

- `method` 字段必须设置为 `startPinpadTransaction`。
- The service must be set to `engine` in the `params`.

该请求包含一个参数列表，第一个参数是服务名称，目前固定为 `engine`，第二个参数是包含所有其他交易相关详细信息的字典。

```json
{
    "jsonrpc":"2.0",
    "method":"startPinpadTransaction",
    "id":"123454352",
    "params": [
        "engine",{
            "vuid": "{{$guid}}",
            "amount":100,
            "tranType":1
        }
    ]
}
```

### 请求参数

下表描述了该请求的参数：

| 姓名         | 类型     | 描述                    |
| :--------- | :----- | :-------------------- |
| `vuid`     | string | 供应商（客户）唯一交易标识符。       |
| `amount`   | 整数     | 交易金额（以分为单位）。          |
| `tranType` | float  | 交易类型代码。例如，`1` 表示标准交易。 |

## 回复

成功的请求将返回一个包含所采集卡片数据的响应，具体内容取决于卡片是通过磁条还是非接触式（EMV）方式读取的。

### 磁条读卡响应

当卡片通过磁条读卡器成功刷卡时，将返回此响应。

```json
{
   "jsonrpc": "2.0",
   "id": "123454352",
   "result": {
       "statusCode": 0,
       "statusMessage": "ok",
       "panEntryMode": 0,
     		"track2": "65458643975398",

   }
}
```
