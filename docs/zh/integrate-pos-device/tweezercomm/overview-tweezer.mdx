TweezerComm 是一种安全高效的支付通信协议，可实现客户端应用程序与 Nayax 人工支付设备之间的无缝集成。它允许销售点 (POS) 系统、电子收银机 (ECR) 和其他支持支付的应用程序通过结构化的 JSON-RPC 接口安全地处理交易。

它使客户端应用程序能够与 Agamento 支付应用程序（运行在 Nayax 有人值守终端上）进行通信，Agamento 是交易处理的中央单元。通信应通过**本地网络**进行，例如**HTTPS** （最佳实践）、 **HTTP** 、**蓝牙**或**USB**连接，从而允许您：

- 发起和管理交易。
- 检索和处理交易数据。
- 处理结算、退款和预授权。

<McpNote />

## 支付服务

Nayax 的 TweezerComm 协议支持两种主要的支付服务，一种面向全球用户，另一种专供以色列用户使用。

- 全球支付：`Engine` Service 是通过 Nayax Gateway 处理信用卡和闭环交易的主要服务。Nayax Gateway 负责设置设备参数，并根据 Nayax Core 中配置的设置处理交易。该服务应用广泛，支持众多国家/地区的多家支付提供商。在执行交易之前，设备必须完成“建立（establishment）”过程（由 Nayax 团队管理）。
- 以色列支付：`Ashrait` Service 是用于执行 AshraitEMV 交易的专用服务。客户端应用程序与 AshraitEMV 应用程序（AEA）进行通信，该应用程序既可在独立终端上运行，也可在中央服务器上运行。所有已批准的交易都会写入 AEA 的数据库。该服务的一个关键部分是批处理过程，即 AEA 将已批准的交易传输到 ABS（SHVA）服务器。
