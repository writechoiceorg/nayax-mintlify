`abortSwipe` 方法用于取消此前通过 `displaySwipeCard` 发起的正在进行的磁卡刷卡操作。

## 要求

<Note>
  JSON-RPC 2.0

  TweezerComm 中的所有方法都遵循 JSON-RPC 2.0 结构。有关请求的结构和发送方式的更多详细信息，请参阅[“入门指南”](/docs/integrate-pos-device/tweezercomm/tweezercomm-get-started) 。
</Note>

要调用此方法，请将 `method` 字段设置为 `abortSwipe`，并按下方所示包含服务名称和参数。

```json
{
    "jsonrpc": "2.0",
    "method": "abortSwipe",
    "id": "123",
    "params": [
        "device", {}
    ]
}
```

### 请求参数

下表描述了该请求的参数：

| 姓名       | 类型       | 描述        |
| -------- | -------- | --------- |
| `device` | `string` | 服务名称（固定值） |

## 回复

成功的请求将返回以下响应：

```json
{
    "jsonrpc": "2.0",
    "id": "123",
    "result": {
        "statusCode": 0,
        "statusMessage": "OK"
    }
}
```

### 响应参数

下表说明了此方法的响应参数：

| 姓名                     | 类型        | 描述           |
| ---------------------- | --------- | ------------ |
| `statusCode`           | `integer` | 操作状态（0 = 成功） |
| `statusMessage`        | `string`  | 指示操作结果的消息    |
