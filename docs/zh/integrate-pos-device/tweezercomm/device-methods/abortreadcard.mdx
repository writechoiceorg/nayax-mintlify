`abortReadCard` 方法用于取消正在进行的 readCard 操作。当用户希望在读卡完成之前停止读卡时，此方法非常有用，例如在超时、用户取消或应用程序逻辑控制的情况下。

## 要求

<Note>
  JSON-RPC 2.0

  TweezerComm 中的所有方法都遵循 JSON-RPC 2.0 结构。有关请求的结构和发送方式的更多详细信息，请参阅[“入门指南”](/docs/integrate-pos-device/tweezercomm/tweezercomm-get-started) 。
</Note>

在请求中，method 字段应设置为 `abortReadCard`，且 params 必须包含固定值 device：

```json
{
    "jsonrpc": "2.0",
    "method": "abortReadCard",
    "id": "123454352",
    "params": [
        "device"
    ]
}
```

### 请求参数

有关此方法可用的请求参数的描述，请参阅下表。

| 姓名       | 类型    | 描述             |
| -------- | ----- | -------------- |
| `params` | array | 必须包含值 `device` |

## 回复

成功的请求将返回类似于以下代码块中的响应：

```json
{
    "jsonrpc": "2.0",
    "id": "123454352",
    "result": {
        "statusCode": 0,
        "statusMessage": "ok"
    }
}
```

<br />

### 响应参数

下面是一个包含所有可能的响应参数及其描述的表格：

| 姓名                     | 类型     | 描述           |
| ---------------------- | ------ | ------------ |
| `statusCode`           | 整数     | 操作状态（0 = 成功） |
| `statusMessage`        | string | 操作结果消息       |
