拆分交易（Split Transaction）流程用于涉及多个子商户的交易。该图展示了 Milgam System 如何在 Nayax 设备上发起一笔交易，并接收包含子零售商特定属性的响应。

## 交易流程

要理解拆分交易流程，请参考以下事件顺序：

<Frame>
  <img src="/images/docs/tweezercomm-split-transaction/image1.png" />
</Frame>

以下是每个步骤的详细说明：

1. The Milgam System sends one or more [`addSubRetailer()`](/docs/integrate-pos-device/tweezercomm/israel-payment-methods/addsubretailer) requests to the Nayax Device. In the example, three such requests are made.
2. The Milgam System performs its [`doPeriodic()`](/docs/integrate-pos-device/tweezercomm/israel-payment-methods/doperiodic) call and then initiates a transaction with a [`doTransaction()`](/docs/integrate-pos-device/tweezercomm/israel-payment-methods/dotransaction-ashrait) request.
3. 持卡人出示其卡片。
4. Nayax 设备与支付服务器通信，对交易执行授权和结算。
5. 支付服务器将交易结果返回给 Nayax 设备。
6. The Nayax Device relays the transaction result to the Milgam System, including a `subRetailer` attribute with the value `XXXXX` to identify the relevant sub-retailer.
