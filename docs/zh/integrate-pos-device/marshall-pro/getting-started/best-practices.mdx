### Request ID

每个请求都必须具有唯一标识符。

- 格式:带连字符的字母数字组合(例如 `refuel-12345`)。
- 上下文:在 ID 中包含业务逻辑,以便日志更易读取。

### 金额处理

VPOS 使用最小货币单位,以避免浮点数误差。

- 如果 `decimalPlace` 为 2,则 $1.00 会以 `100` 的形式发送。
- 如果 `decimalPlace` 为 3,则 $1.00 会以 `1000` 的形式发送。

### 常规超时时间

客户端系统应管理超时时间,以确保终端不会无限期挂起:

| 阶段   | 建议超时时间   |
| :--- | :------- |
| 卡片检测 | 30–60 秒  |
| 用户输入 | 60–120 秒 |
| 授权   | 10–30 秒  |

### 错误处理

- 日志记录:请务必存储 `errorCode` 和 `errorMessage`。
- 重试:请勿重试被拒绝(DECLINED)的交易。拒绝是银行给出的最终响应;用户必须提供其他支付方式。
