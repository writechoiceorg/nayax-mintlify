`info` 方法用于在客户端应用程序(POS/FCC)与 VPOS 终端之间交换版本信息和设备元数据。这通常是在建立 TCP 连接后发送的第一个命令,以确保两个系统之间的兼容性。

使用此方法可以：

- 交换供应商、型号和序列号等信息以进行系统识别。
- 同步软件和硬件版本，以确保协议功能一致。
- 检索关键终端设置，例如交易金额定义的小数位数。

## 请求

所有请求都必须包含 `command` 和唯一的 `requestId`。`data` 对象应包含发起请求的客户端系统的详细信息。

以下示例展示了一个典型的请求：

```json
{
"command": "info",
"requestId": "req-020",
"data": {
"vendor": "Nayax Fuel",
"model": "FCC-3000",
"serialNumber": "FCC-12345678",
"swVersion": "1.2.3.4",
"hwVersion": "4.3.2.1"
}
}
```

### 参数定义

下表描述了您的系统在此请求中提供的参数：

| 姓名   | 类型     | 必需的 | 描述                            |
| :--- | :----- | :-- | :---------------------------- |
| 小贩   | string | 是的  | 客户端系统的供应商名称（例如，“Nayax Fuel”）。 |
| 模型   | string | 是的  | POS机或FCC硬件的型号标识符。             |
| 序列号  | string | 是的  | 客户端系统的唯一序列号。                  |
| 软件版本 | string | 是的  | 客户端应用程序的当前软件版本。               |
| 硬件版本 | string | 是的  | 客户端设备的硬件版本。                   |

## 回应

VPOS终端会返回自身的硬件和软件规格，以及支付处理所需的区域设置。以下示例显示了典型的响应：

```json
{
"requestId": "req-020",
"data": {
"serialNumber": "P129901X251AD",
"osVersion": "16.0",
"appVersion": "2.3.4.5",
"paymentVersion": "0.0.2.68",
"decimalPlace": 2
}
}
```

### 响应参数

以下是VPOS终端返回的参数：

| 姓名     | 类型     | 描述                                             |
| :----- | :----- | :--------------------------------------------- |
| 序列号    | string | VPOS终端的唯一序列号。                                  |
| 操作系统版本 | string | 设备上运行的操作系统版本。                                  |
| 应用版本   | string | 主终端应用程序的版本。                                    |
| 支付版本   | string | 支付处理模块的具体版本。                                   |
| 小数位    | 整数     | 用于所有交易金额的小数位数(例如,`2` 表示金额 `5000` 代表 `$50.00`)。 |
