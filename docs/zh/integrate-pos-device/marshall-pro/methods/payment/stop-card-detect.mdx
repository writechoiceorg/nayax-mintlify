`stopCardDetect` 方法是一个实用命令,用于手动停用终端的读卡器。当交易被放弃或需要执行其他工作流程时,这有助于控制用户体验。

使用此方法可以：

- 如果客户在加油机或 POS 上取消操作,则终止检测阶段。
- 如果在没有用户交互的情况下达到本地管理的超时时间,请安全关闭读卡器。
- 清除读卡器状态,以便终端准备好执行其他命令或返回空闲屏幕。

Nayax 设备上停止检测屏幕的示例:

<Frame>
  停止卡检测
</Frame>

## 请求

此请求是一个简单的触发器,不需要 `data` 对象。您必须提供标准的 `command` 和唯一的 `requestId`。

以下示例展示了一个典型的请求：

```json
{
"command": "stopCardDetect",
"requestId": "req-004"
}
```

### 参数定义

下表描述了停止请求所需的字段:

| 姓名    | 类型     | 必需的 | 描述                     |
| :---- | :----- | :-- | :--------------------- |
| 命令    | string | 是的  | 设置为`"stopCardDetect"`。 |
| 请求 ID | string | 是的  | 用于跟踪响应的唯一标识符。          |

## 回应

读卡器成功禁用后,终端将确认该命令。

以下示例展示了一个典型的响应：

```json
{
"requestId": "req-004"
}
```

### 响应参数

响应确认检测会话已终止:

| 姓名    | 类型     | 描述            |
| :---- | :----- | :------------ |
| 请求 ID | string | 返回原始请求的唯一标识符。 |
