`print` 方法可直接访问 VPOS 终端的集成热敏打印机。这使客户端应用程序能够为客户生成物理记录，例如交易收据、忠诚度优惠券或加油汇总单。

使用此方法可以：

- 打印最终销售详情，包括商户信息、已屏蔽的银行卡数据和交易收据总额。
- 喷嘴收回后，生成体积、等级和单价的实物记录。
- 根据顾客当前的购买情况，向其提供折扣码或奖励券。
- 直接在工作站打印诊断或状态报告，用于维护日志。

## 请求

请求包含您希望打印的原始文本内容或结构化数据。您可以使用标准转义字符进行基本格式设置，例如换行。

以下示例展示了一个典型的请求：

```json
{
"command": "print",
"requestId": "req-012",
"data": {
"printData": "Transaction Receipt\n-------------------\nAmount: $20.00\nDate: 2026-01-
13\nApproved"
}
}
```

### 参数定义

下表描述了此请求可用的参数：

| 姓名    | 类型     | 必需的 | 描述                                  |
| :---- | :----- | :-- | :---------------------------------- |
| 命令    | string | 是的  | 设置为 `"print"`。                      |
| 请求 ID | string | 是的  | 用于跟踪响应的唯一标识符。                       |
| 内容    | string | 是的  | 要打印的文本数据。使用 `\n` 表示换行。              |
| 剪纸    | 布尔值    | 不   | 如果为 `true`，则在打印内容后打印机将执行全切（如果硬件支持）。 |

## 回应

打印作业成功排队或由硬件完成后，终端会返回确认信息。

以下示例展示了一个典型的响应：

```json
{
"requestId": "req-012",
"data": {
"printed": true,
"characterCount": 71
}
}
```

### 响应参数

该响应确认了外围动作的状态：

| 姓名         | 类型     | 描述                                              |
| :--------- | :----- | :---------------------------------------------- |
| 请求 ID      | string | 返回原始请求的唯一标识符。                                   |
| **status** | string | 如果打印任务被接受，返回 `SUCCESS`；如果打印机缺纸或未连接，则返回 `ERROR`。 |
