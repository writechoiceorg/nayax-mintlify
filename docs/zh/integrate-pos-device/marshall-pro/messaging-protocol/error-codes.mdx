本页提供错误代码参考。所有错误共享相同的响应结构,并在 `errorCode` 和 `errorMessage` 字段中返回。

### 协议错误

| 代码   | 意义                    |
| :--- | :-------------------- |
| 1001 | 无效的 JSON              |
| 1002 | 未知命令                  |
| 1003 | 缺少必需参数                |
| 1004 | 参数值无效                 |
| 1005 | `displayInput` 提示签名无效 |

### 错误处理最佳实践

为确保高质量的集成,请遵循以下准则:

- 日志记录:请务必同时记录 `errorCode` 和描述性的 `errorMessage`,以便于调试。
- 重试:如果终端未在预期时间窗口内响应,请在客户端实现重试逻辑。
- 安全性:切勿在没有用户直接操作的情况下自动重试被拒绝的交易。
- 典型超时时间:请使用以下时间窗口进行客户端监控:
  - 卡片检测:30–60 秒。
  - 用户输入:60–120 秒。
  - 授权:10–30 秒。

<Note>
  用户反馈
  在等待期间或发生错误时,请务必在 POS 屏幕上提供清晰的视觉反馈,以便让客户随时了解情况。
</Note>
