为了确保每笔交易的安全性和完整性，Spark API 要求采用双重身份验证流程。所有请求都必须标识发送者，并包含数字签名，以验证请求体未被篡改。

## 身份验证选项

选择与项目状态相符的集成路径：

  <Card icon="link" title="验证" href="/docs/integrate-pos-device/spark/security-authentication/spark-authentication">
    这是所有新集成的当前标准。此版本使用简化的基于标头的签名。
  </Card>

  <Card icon="link" title="传统身份验证" href="/docs/integrate-pos-device/spark/security-authentication/spark-legacy-authentication">
