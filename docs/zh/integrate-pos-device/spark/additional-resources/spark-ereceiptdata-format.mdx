Nayax eReceipt 平台可实现与数字收据提供商的无缝集成，使其能够向 Nayax 客户提供数字收据服务。在第三方服务器上持有额外交易信息的集成方可以使用 `eReceiptData` 字段将这些数据包含在消费者的数字收据中。

`eReceiptData` 字段目前仅适用于 `/Settlement` 方法中的预授权（Pre-Authorization）流程。预选（Pre-selection）尚不支持此功能。

该字段中提供的任何信息都将直接显示在消费者的电子收据（eReceipt）上。因此，避免共享敏感或不相关的数据至关重要。

## 格式规范

`eReceiptData` 字段需要一个 JSON 对象。该对象允许将数据组织为不同的区块，每个区块包含一个键值对对象数组。

```json
{
  "sectionName1": [
    {"key": "value", "key": "value"}
  ],
  "sectionName2": [
    {"key": "value", "key": "value", "key": "value", "key": "value"}
  ],
  "sectionName3": [
    {"key": "value", "key": "value", "key": "value", "key": "value"}
  ]
}

```

如果您不希望使用特定的区块名称，可以将 `sectionName` 留空。字段将按照 JSON 结构中提供的顺序显示在数字收据上。
