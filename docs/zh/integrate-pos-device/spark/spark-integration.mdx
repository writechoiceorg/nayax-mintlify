与 Spark 集成使您的后端系统能够通过安全的、由 API 驱动的工作流程来控制 Nayax 的支付终端。Nayax 提供了结构化的入驻流程、认证清单和生产环境上线计划，以确保实施稳定合规。

本页概述了 Nayax 如何通过集成支持合作伙伴，并解释了从开始到结束所涉及的步骤。

## Nayax 提供什么

Nayax 在启动期间提供以下资源和材料：

- OpenAPI YAML 文件包含所有可用端点的完整规范。
- 示例代码
- 认证清单用于验证您的 Spark 实施。
- 沙盒凭证，包括 API 令牌、签名密钥和测试环境 URL。
- 支持渠道，例如 Microsoft Teams 或来自 Nayax 集成工程师的电子邮件支持。

<Check>
  开放API

  将 OpenAPI YAML 与 Postman 或 Swagger UI 等工具结合使用，可以在开发过程中快速测试您的请求。
</Check>

## 集成过程

Nayax 采用分阶段的方法来帮助开发人员集成 Spark：

1. **启动**：您与 Nayax 会面以审查集成范围并接收文档。
2. **沙盒设置**：Nayax 为您的团队提供测试环境和凭证。
3. **实施**：您可以使用提供的工具和指南与 Spark API 集成。
4. **认证**：Nayax 根据测试清单审查您的实施情况。
5. **生产部署**：一旦获得认证，您将收到生产密钥并上线。
