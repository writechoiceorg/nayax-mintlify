`TriggerTransaction` 端点用于发起与物理终端的用户交互。它会向指定设备发送唤醒信号。请求应包含一个值，Nayax 将使用该值对消费者的支付卡进行授权（预授权）或收费（预选）。

- 如果实现了预授权（Pre-Authorization）流程，则可以从 Nayax Core 管理和检索授权值。
- 如果实现了预选（Pre-Selection）流程，缺少金额的请求将返回错误。

## 要求

集成方向 `/TriggerTransaction` 端点发送 POST 请求。正文参数需如下例所示：

```json
{
  "SparkTransactionId": "12c7cec2-c690-4425-9a1f-db0db60e2d8c",
  "TerminalId": "0434334921100366",
  "TerminalIdType": 1,
  "Amount": 15.00,
  "PosDisplay": ";Charging Station:3;Day:30.52p;Night:9p;Standing charge:52.18p/day; Off-peak:12:30-04:30;",
  "TransactionTimeout": 60
}
```

## 回复

成功的响应将返回以下正文：

```json

{
 "SparkTransactionId": "12c7cec2-c690-4425-9a1f-db0db60e2d8c",
 "Status": {
 "Verdict": "Approved",
 "ErrorDescription": " No Errors "
 }
}
```

### 响应参数

下表说明了响应的参数：

| 场地                        | 类型     | 描述                                      |
| :------------------------ | :----- | :-------------------------------------- |
| `SparkTransactionId`      | String | 回显请求中的交易 ID                             |
| `Status`                  | Object | 包含最终判定结果和可选的错误描述                        |
| `Status.Verdict`          | String | 请求的结果（`Approved`、`Rejected` 等）          |
| `Status.ErrorDescription` | String | 提供发生的任何错误的人类可读描述；若请求成功，则显示 `No Errors`。 |
| `Status.ErrorCode`        | 数字     | 指示交易期间发生的特定错误类型的数字代码。此字段仅出现在错误响应中。      |
