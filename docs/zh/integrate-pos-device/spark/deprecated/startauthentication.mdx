`StartAuthentication` 端点是任何 Spark 交易的第一步。它使用预共享令牌和经过加密签名的负载对交易请求进行身份验证。

Spark 会返回一个唯一的 `HashedSparkTransactionId`，该 ID 必须用于后续所有步骤。

## 要求

集成方向 `/spark/StartAuthentication` 端点发送 POST 请求。正文参数需如下例所示：

```json
{
 "TokenId": 116383,
 "TerminalId": "0434334921100366",
 "TerminalIdType": 1,
 "Random": "123456789qwertyui",
 "Cipher": "X305dITNTAw2vHsxE+taVcn6UvgBC3fdI6QbqeABgHbo8CKsoZhqISJfslehCiA+L7XYrqvKFci7C6BNj/trzBuNJwBEjgBzKhhgpJ5ggnw="
}
```

<Warning>
  安全

  This request must be signed with the `Signature` and `IntegratorId` headers. See [Security & Authentication](/docs/integrate-pos-device/spark/security-authentication/spark-authentication) to learn more.
</Warning>

### 请求参数

下表描述了该请求的参数：

| 场地               | 类型     | 必需的 | 描述                                                                                                                   |
| ---------------- | ------ | --- | -------------------------------------------------------------------------------------------------------------------- |
| `TokenId`        | 整数     | 是的  | Nayax 提供的用于标识您的集成的唯一令牌                                                                                               |
| `TerminalId`     | String | 是的  | 目标终端的唯一设备 ID                                                                                                         |
| `TerminalIdType` | 整数     | 是的  | 定义所使用的 ID 类型。`1` 表示硬件序列号（HW Serial），`2` 表示 Nayax 机器 ID。                                                              |
| `Random`         | String | 是的  | 用作加密握手一部分的随机字符串                                                                                                      |
| `Cipher`         | String | 是的  | AES-256 ECB encrypted payload using Token. See the [section below](#build-cipher) to learn how to build this cipher. |

### Build `Cipher`

您的密文由以下部分组成：

- 一个 36 个字符的 Spark 事务 ID（一个带有连字符的 GUID）。
- 单个 "=" 字符。
- 一个 17 个字符的字母数字随机字符串。
- 一个 10 个字符的 `YYMMDDhhmm` 格式 UTC 时间戳。

请按照以下步骤生成并加密您的密文：

1. 将 Spark 事务 ID、“=”字符、随机字符串和时间戳连接起来，形成 64 个字符的密文，如下例所示：
   ```
   12c7cec2-c690-4425-9a1f-db0db60e2d8c=123456789qwertyui2306061021
   ```
2. 从提供的令牌中提取最右侧的 32 个字符，以创建 256 位 AES 加密密钥。例如，假设令牌如下：
   ```
   some_long_token_wRvTVTkungMIKThTVbj_fiXdfoGclhn0
   ```
   加密密钥如下：
   ```
   wRvTVTkungMIKThTVbj_fiXdfoGclhn0
   ```
3. 使用 ECB 模式的 AES 加密 64 个字符的密文。确保明文已填充（例如，使用 PKCS7）至 16 字节的倍数。对生成的加密字节进行 Base64 编码以进行传输，这将产生类似于以下示例的输出：
   ```
   X305dITNTAw2vHsxE+taVcn6UvgBC3fdI6QbqeABgHbo8CKsoZhqISJfslehCiA+L7XYrqvKFci7C6BNj/trzBuNJwBEjgBzKhhgpJ5ggnw=
   ```

## 回复

成功的响应将返回以下正文：

```json
{
 "HashedSparkTransactionId": "9515af0d6b5c328b6383ff241fa7b03e489d6285a8db6a4f0d8369e3f044e8d7",
 "Status": {
   "Verdict": "Approved",
   "ErrorDescription": " No Errors "
 }
}
```

### 响应参数

下表说明了响应的参数：

| 参数名称                       | 类型     | 描述                                              |
| -------------------------- | ------ | ----------------------------------------------- |
| `HashedSparkTransactionId` | String | 请求中发送的 Spark 交易 ID 的 SHA256 哈希值。                |
| `Status`                   | Object | 一个包含交易结果详情的对象，包括判定结果以及任何相关的错误描述或错误代码。           |
| `Status.Verdict`           | String | 表示身份验证尝试的总体结果。成功时可为 `Approved`，失败时为 `Declined`。 |
| `Status.ErrorDescription`  | String | 提供发生的任何错误的可读描述，若请求成功则显示 `No Errors`。            |
| `Status.ErrorCode`         | 数字     | 指示交易期间发生的特定错误类型的数字代码。此字段仅出现在错误响应中。              |
