  og:description: 预授权支付是 Nayax 推荐的方式,因为其流程更为稳健,因而更加安全。该流程由两个步骤组成。

Nayax 推荐使用预授权方式。它将支付分为两个步骤：

1. 预授权
2. 沉降

<Note>
  如果您愿意，可以一步完成授权和结算。要了解更多有关此流程的信息，请访问[“一步付款”](/docs/integrate-pos-device/emv-core/payment-flows/single-step-payment)页面。
</Note>

## 预授权

第一步，主机使用 EMV 核心对买家的卡进行指定金额的支付操作进行授权。

下图显示了**预授权**的交易流程：

<Frame>
  <img src="/images/docs/payment-with-authorization/image1.png" />
</Frame>

在哪里：

1. The host machine calls the [`PreAuthorize`](/docs/integrate-pos-device/emv-core/sdk-methods-functions/payment-1#preauthorize) method using the EMV Core API. This is the only action performed by the host machine.
2. EMV核心会轮询读卡器。您的应用程序应提示用户轻触他们的卡。
3. 读卡器提示用户出示银行卡。
4. 用户刷卡。
5. 读卡器将加密的卡片数据发送到 EMV Core。
6. EMV Core 将数据发送到 Nayax 服务器，以授权步骤 1 中的金额。
7. Nayax 服务器返回包含交易 ID 和数据的授权响应。
8. EMV Core 会触发 `TransactionComplete` 事件,并附带交易 ID 和结果。

<Note>
  无论结果如何——批准、拒绝或超时——`TransactionComplete` 回调都会触发。请检查交易数据以确定结果并决定后续操作。
</Note>

交易获批后，即可继续进行销售流程：交付产品、开始收费或执行其他操作。您还可以将交易 ID 和数据存储在服务器上。

## 沉降

结算流程确定最终金额并从买方的信用卡中扣除。结算流程在产品交付或操作（例如电动汽车充电）完成后开始。

<Frame>
  <img src="/images/docs/payment-with-authorization/image2.png" />
</Frame>

1. 销售操作完成。例如，产品已交付给买方。
2. 您的应用程序会将销售情况报告给您的应用程序服务器。
3. 应用服务器关闭会话。
4. Your application calls [`ConfirmTransaction`](/docs/integrate-pos-device/emv-core/sdk-methods-functions/payment-1#confirmtransaction) with the transaction ID and the final amount to charge.

<Note>
  最终金额可能与预授权金额不同。为避免出现问题，如果最终金额可能有所变动，请在授权时使用预计最高金额。
</Note>

5. EMV Core 确认金额并向 Nayax 服务器发起结算。此时结算尚未完成。
6. EMV Core 将交易 ID 和金额发送到 Nayax 服务器。
7. Nayax 服务器会返回确认信息，因此 EMV Core 不需要重新发送确认信息。

Nayax负责处理结算并转移资金。

<Note>
  要取消预授权交易，请发送撤销操作而不是确认操作。
</Note>
