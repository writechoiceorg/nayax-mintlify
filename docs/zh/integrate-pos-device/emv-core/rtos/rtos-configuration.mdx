配置参数由 Nayax Core 集中管理，可能因客户和机器而异。

## 地点

配置文件保存在读卡器文件系统的 `E:\ETC` 文件夹中。

<Frame>
  <img src="/images/docs/rtos-configuration/config-file-location.png" />
</Frame>

## 格式

```
<Parameter Name>=<Value>
```

示例：`TMS_SVC_URL=tms.otiglobal.com`

## 管理设置

配置由 **Nayax Core**集中管理。要调整设置：

1. 导航至**操作** > **机器**。
2. 找到**“嵌入式”**和**“嵌入式高级”**部分。
3. 更新相关属性并点击 **保存**。服务器会自动生成配置文件并将其推送至设备。

## 网络和系统设置

这些参数控制信号接口选择和系统级行为：

| 范围                      | 描述                                       | 类型       | 默认 | 自版本以来。 |
| :---------------------- | :--------------------------------------- | :------- | :- | :----- |
| `ETH_INTERFACE`         | 网络接口；空 = 自动检测，如果检测到则回退到 eth0             | String   | 空的 | —      |
| `KIOSK_USE_TCP_SOCKETS` | 请使用 TCP 端口 10000（命令）和 10001（事件）代替域套接字。   | 布尔值（1/0） | 0  | —      |
| `DAILY_RESTART_HOUR`    | 每日自动重启时间（0-23，0 = 关闭）。实际重启时间为设定时间后30分钟内。 | U8       | 0  | —      |
| `CONFIG_VERSION`        | 配置文件版本；已发送至TMS，设备未使用。                    | String   | —  | —      |
| `KIOSK_TCP_LOCALHOST`   | 限制 EMV Core 套接字仅限本地主机接口                  | 布尔值（1/0） | 0  | 0.7    |

## TMS 和支付服务设置

这些参数用于配置TMS和支付服务器的URL和轮询行为：

| 范围                     | 描述                                                                                     | 类型               | 默认                | 自版本以来。    |
| :--------------------- | :------------------------------------------------------------------------------------- | :--------------- | :---------------- | :-------- |
| `TMS_SVC_URL`          | TMS 服务的 URL。使用 `tms.otiglobal.com` 表示 OTI TMS，或使用 `tms-c.otiglobal.com` 表示 Nayax Core。 | URL字符串           | tms.otiglobal.com | 0.7 / 1.0 |
| `PAYMENT_SVC_URL`      | 支付服务器 URL，仅名称（例如 `payment-nayax-p.otiglobal.com`）                                      | URL字符串（最多256个字符） | 空的                | —         |
| `SERVER_POLL_INTERVAL` | 服务器请求更新的间隔时间（秒）。数值越低，响应速度越快，但数据使用量越大。                                                  | U16              | 600               | —         |

## 阅读器设置

这些参数控制初始化命令、按钮行为和阅读器显示消息：

| 范围                        | 描述                                                                         | 类型         | 默认  | 自版本以来。    |
| :------------------------ | :------------------------------------------------------------------------- | :--------- | :-- | :-------- |
| `INIT_CMD`                | 初始化时发送给 Saturn Reader 的命令。以 `\|` 分隔的 TLV 标签十六进制字符串。示例：`DF0A0100\|DF7A0100` | 十六进制字符串    | 空的  | —         |
| `STOP_BUTTON_ID`          | 停止/取消按钮的 ID：1 = 右，2 = 左                                                    | U8         | 1   | —         |
| `STOP_BUTTON_LED_PRESENT` | 0 = 关闭按钮闪烁（如果 LED 指示灯未亮起，则表示 STOP 按钮未亮起）                                   | 布尔值（1/0）   | 1   | —         |
| `R_COUNTRY_CODE`          | 设置读卡器中的终端国家/地区代码值。v1.0 版本从 Nayax Core 机器的国家/地区获取该值。                        | 字符串（4 个字符） | 不适用 | 0.7 / 1.0 |
| `R_DO`                    | 在设备启动时向读卡器发送 DO 命令。与 `INIT_CMD` 不同，此命令仅执行一次。多行 `R_DO` 会全部执行。               | 十六进制字符串    | 不适用 | —         |
| `R_MSG_X_Y`               | 设备启动时设置读取器消息（索引 X，行 Y）                                                     | ASCII 字符串  | 不适用 | —         |

## 安全设置

这些参数控制服务器连接的 SSL 和加密设置：

| 范围                      | 描述                                                              | 类型       | 默认 | 自版本以来。 |
| :---------------------- | :-------------------------------------------------------------- | :------- | :- | :----- |
| `TMS_USE_SSL`           | 启用或禁用 TMS 服务的 SSL                                               | 布尔值（1/0） | 0  | —      |
| `PMT_DUKPT_KEY_VARIANT` | DUKPT密钥变体取决于PSP型号。使用十进制格式（例如，十进制17 = 0x11）。设置为0可禁用DUKPT（仅用于测试）。 | U8       | 17 | —      |
| `PAYMENT_SVC_USE_SSL`   | 开启或关闭支付服务的 SSL                                                  | 布尔值（1/0） | —  | —      |

## 开放式支付设置

这些参数控制开环（信用卡/借记卡）支付行为、超时时间和授权流程：

| 范围                                  | 描述                                                                                                                                                                            | 类型       | 默认   | 自版本以来。    |
| :---------------------------------- | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :------- | :--- | :-------- |
| `PMT_ACCEPT_OFFLINE`                | 存储转发：网络不可用时自动批准交易。 **注意：** 存在交易未经批准的风险；运营商对此负有责任。                                                                                                                             | 布尔值（1/0） | 0    | —         |
| `PMT_KIOSK_AUTOCONFIRM`             | 如果会话中断，则自动使用预授权金额进行确认。支持长时间会话，并可进行重置（例如，电动汽车充电器）。                                                                                                                             | 布尔值（1/0） | 1    | —         |
| `PMT_CONFIRMATION_INTERVAL`         | 两次尝试执行待处理的确认或作废操作之间的秒数                                                                                                                                                        | U32      | 5    | —         |
| `PAYMENT_SVC_SERVER_RETRIES`        | 在线授权出错后会重试多少次？                                                                                                                                                                | 整数       | 3    | —         |
| `PAYMENT_SVC_SERVER_TIMEOUT`        | 支付服务器请求的HTTP超时时间（秒）                                                                                                                                                           | 整数       | 20   | —         |
| `PAYMENT_SVC_CONFIRM_TIMEOUT`       | 发送确认或取消请求时的服务器超时时间（秒）                                                                                                                                                         | U32      | 60   | —         |
| `PAYMENT_SVC_USER_WAITING_MAX_TIME` | 在线授权的最大等待时间（秒）                                                                                                                                                                | U16      | 30   | —         |
| `MAESTRO_LIMIT_CENTS`               | 使用 Maestro 卡进行超过此限额的交易将被拒绝，且没有备用方案。                                                                                                                                           | U32      | 2500 | —         |
| `PAYMENT_SVC_SYNCHRONOUS`           | Wait for the current transaction to close on the server before starting a new one. Note: for asynchronous mode after an unexpected reset, also set `PMT_KIOSK_AUTOCONFIRM=0`. | 布尔值（1/0） | 0    | —         |
| `OPEN_LOOP_ENABLED`                 | 启用/禁用开放式支付                                                                                                                                                                    | 布尔值（1/0） | 1    | 0.7       |
| `PRE_AUTH_AMOUNT`                   | 如果 API 未提供金额，则默认预授权金额为美分。                                                                                                                                                     | 整数       | 0    | 0.7       |
| `CURRENCY_CODE`                     | ISO 4217 代码（例如，840 = 美元）。v1.0 版本会在每次机器属性更改或执行“重新加载配置”操作后自动从 Nayax Core 获取该代码。                                                                                                 | String   | 840  | 0.7 / 1.0 |

## 闭环支付设置

这些参数用于启用和配置闭环（预付费）卡支持，包括 Mifare 卡：

| 范围                               | 描述                                                                                               | 类型                                                               | 默认     | 自版本以来。 |   |
| :------------------------------- | :----------------------------------------------------------------------------------------------- | :--------------------------------------------------------------- | :----- | :----- | - |
| `CLP_PROPRIATARY_TRACKS_ENABLED` | 启用EMV支付过程中专有磁道的闭环检测功能。需要正确配置读卡器。                                                                 | 布尔值（1/0）                                                         | 0      | —      |   |
| `CLP_MIFARE_READ`                | 指定要读取的 Mifare 数据块。格式：\`K\[A                                                                      | B]\[key\_num]B\[block\_num],...`Example:`KA01B10B14,KB05B20B30\` | String | —      | — |
| `CLP_MIFARE_ENABLED`             | 在支付期间将 Mifare 卡作为闭环卡启用。0 = 禁用；1 = 通过服务器启用；2 = 通过本地主机 API 启用（UID 会立即在 `TransactionComplete` 中返回）。 | 整数                                                               | 0      | 0.7    |   |

## 日志记录设置

这些参数控制日志文件大小、保留时间和每个模块的详细程度：

| 范围                    | 描述                                              | 类型  | 默认  | 自版本以来。 |
| :-------------------- | :---------------------------------------------- | :-- | :-- | :----- |
| `LOG_HOURS_TO_UPLOAD` | UPLOAD\_LOG 命令中要包含的日志小时数                        | U16 | 10  | —      |
| `LOG_NO_OF_FILES`     | 保留的日志文件数量。当所有文件都达到 `LOG_FILE_SIZE` 时，最旧的文件会被覆盖。 | 整数  | 6   | —      |
| `LOG_FILE_SIZE`       | 每个日志文件的最大大小（以 KB 为单位）                           | 整数  | 256 | —      |
| `LOG_GLOBAL`          | 所有模块的默认日志级别（1-5）                                | 1–5 | 3   | —      |
| `LOG_READER`          | 读取器日志级别                                         | 1–5 | 3   | —      |
| `LOG_ETH`             | 以太网日志级别（仅限错误；数据请使用 LOG\_NET）                    | 1–5 | 3   | —      |
| `LOG_NET`             | oT\_Net（网络）的日志级别                                | 1–5 | 3   | —      |
| `LOG_SYSTEM`          | 系统或通用功能的日志级别                                    | 1–5 | 3   | —      |
| `LOG_APP_3`           | 开环日志级别                                          | 1–5 | 3   | —      |
| `LOG_APP_4`           | TMS 的日志级别                                       | 1–5 | 3   | —      |
| `LOG_USBH2`           | USB 主机的日志级别                                     | 1–5 | 3   | —      |

## 读者消息定制

这些参数可以自定义 Saturn Reader 在不同状态下显示的文本：

| 范围                             | 描述            | 类型      | 默认      |
| :----------------------------- | :------------ | :------ | :------ |
| `SAT_MSG_DECLINED1`            | “已拒绝”消息的第一行   | 字符串（16） | —       |
| `SAT_MSG_DECLINED2`            | “已拒绝”消息的第 2 行 | 字符串（16） | —       |
| `SAT_MSG_ERROR1`               | “错误”消息的第一行    | 字符串（16） | —       |
| `SAT_MSG_ERROR2`               | “错误”消息的第 2 行  | 字符串（16） | —       |
| `SAT_MSG_CARD_NOT_RECOGNIZED1` | 第 1 行，用于闭环支付  | 字符串（16） | 卡片未     |
| `SAT_MSG_CARD_NOT_RECOGNIZED2` | 第 2 行，用于闭环支付  | 字符串（16） | 认可      |
| `SAT_MSG_CLP_AUTHORIZING1`     | 第 1 行，用于闭环支付  | 字符串（16） | 正在授权... |
| `SAT_MSG_CLP_APPROVED1`        | 第 1 行，用于闭环支付  | 字符串（16） | 得到正式认可的 |
| `SAT_MSG_CLP_APPROVED2`        | 第 2 行，用于闭环支付  | 字符串（16） | 平衡：     |
| `SAT_MSG_SWIPE_ALL1`           | 所有通道中重试交易的消息  | 字符串（16） | 插入或刷卡   |
| `SAT_MSG_SWIPE_ALL2`           | —             | 字符串（16） | 或兑换卡    |
| `SAT_MSG_DEFAULT_MSGSWIPE1`    | 通过磁条卡重试交易的消息  | 字符串（16） | 请滑动     |
| `SAT_MSG_DEFAULT_MSGSWIPE2`    | —             | 字符串（16） | 卡片      |
