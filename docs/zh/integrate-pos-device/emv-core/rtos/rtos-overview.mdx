EMV Core RTOS SDK 是一个可移植的 C99 库，用于为基于 MCU 和 RTOS 的硬件添加 EMV 支付功能。您的应用程序调用 SDK API；SDK 负责处理读卡、授权和服务器通信。

<Note>
  本文档涵盖适用于 MCU 和 RTOS 平台的 SDK v1.0。有关 Android 或 Linux SDK，请参阅[Android 安装](/docs/integrate-pos-device/emv-core/get-started/emv-core-android-installation)或[Linux 安装](/docs/integrate-pos-device/emv-core/get-started/emv-core-linux-installation)。
</Note>

## SDK亮点

- 专为嵌入式系统设计的**C99 源代码**
- **完整的EMV交易处理**，包括命令处理和授权
- **平台抽象层，**无需修改代码即可在任何实时操作系统上运行。
- SDK 中包含的**Linux 参考实现**
- 用于快速测试您的设置的**示例应用程序**

### 应用层

信息传递过程遵循以下几个层次：

| 层          | 描述                                                  |
| :--------- | :-------------------------------------------------- |
| 应用层        | 您的代码调用 SDK API(例如 `EcodSdk_PreAuthorize()`)         |
| JSON-RPC 层 | 将 API 调用转换为带有数据链路帧的 JSON-RPC 消息                     |
| 串行传输       | 通过 UART(`/dev/ttyUSB0`)或 HID(`/dev/hidraw0`)发送成帧的消息 |
| EMV核心设备    | 处理付款请求并返回响应                                         |
| 回调系统       | 通过已注册的回调函数传递异步响应                                    |

## 需求概述

| 要求     | 细节                             |
| :----- | :----------------------------- |
| 设备连接   | USB HID 或 UART TTL             |
| 最低内存要求 | 64 KB（可配置以适应较小的系统）             |
| 读者力量   | 5-12 VDC,空闲 0.7 W / 峰值 3 W     |
| 网络     | IPv4 TCP 套接字、DNS 解析、仅客户端       |
| 编译器    | 支持 C99 的 GCC 4.8+ 或 Clang 3.5+ |
