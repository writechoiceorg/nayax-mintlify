### 包装内容

每个平台文件夹均包含四个主要组件:

- `otiKiosk`:主要的 EMV Core 服务可执行文件。
- `otiKioskDemo`:客户端测试应用程序可执行文件。
- `kiosk-client-library-xxx.zip`:包含 C 源代码和示例应用程序的 Eclipse 项目。
- `otikiosk-xxx.zip`:用于通过 TMS 进行远程更新的软件包(使用 `CON_PROG` 命令)。

该软件包的组织结构如下:

```text
Linux/
└── x86_64/
    ├── otiKiosk/            # EMV Core Service
    ├── otiKioskDemo/        # Test App
    ├── Sources/             # Client library & source code
    └── TMS Update/          # Remote update package
```

## 安装

适用于 Linux 的 EMV Core SDK 提供了一组 API，用于通过 Unix 域套接字或 TCP 控制支付流程。它支持多种架构，包括 x86\_64 和各种 ARM 平台。

### 先决条件与环境

在部署 SDK 之前,请确认以下要求:

- 系统依赖项:需要 `libcurl` 以支持 TLS 1.2 或更高版本的安全互联网访问。
- 权限:运行该服务的用户必须对可执行文件所在文件夹及读卡器设备(例如 USB 对应的 `/dev/hidrawX` 或串口对应的 `/dev/ttyXXX`)具有读写权限。
- 架构:请确保为目标平台使用正确的二进制文件(例如 Ubuntu x86\_64、armhf 或 armv8a)。

### 安装步骤

1. 将 EMV Core 二进制文件(`otiKiosk`)复制到目标设备。请注意,不得重命名该文件。
2. 构建示例应用程序:
   - 安装构建基础工具:`sudo apt-get install build-essential automake`。
   - 解压源代码压缩文件(例如 `Emv-Core-client-x86_64-1.4-97.zip`)。
   - 在源代码文件夹中运行以下命令:
     ```bash
     autoreconf -fi
     ./configure
     make
     ```
3. 运行应用程序:
   - 终端 1:使用 `sudo ./otiKiosk` 启动 EMV Core 服务。
   - 终端 2:使用 `sudo ./otiKioskDemo` 启动测试应用程序。

<Note>
  通信配置

  By default, the SDK uses Internal (Unix domain) sockets, creating `socket_cmd` and `socket_events` in the application's `/var` folder.

  If you prefer TCP communication, set the configuration option `KIOSK_USE_TCP_SOCKETS=1`. This will use ports **10000** for commands and **10001** for events.
</Note>
