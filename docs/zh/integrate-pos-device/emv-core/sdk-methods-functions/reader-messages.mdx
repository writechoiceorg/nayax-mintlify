读卡器消息对于在交易过程中向用户传达读卡器状态至关重要。EMV Core SDK 提供了 `ReaderMessageEvent` 回调来转发这些消息，这些消息可以显示在应用程序的用户界面中。

以下部分包含描述可以显示的不同阅读器消息的表格。

## 主要读卡器消息

以下是针对读卡器主要运行状态、状态转换和错误情况发送的消息：

| 类别    | 十六进制（内部） | 十进制（API） | 消息详细信息                              |
| :---- | :------- | :------- | :---------------------------------- |
| 主要州   | 0x00     | 0        | （空白屏幕）                              |
|       | 0x01     | 1        | 礼品卡                                 |
|       | 0x11     | 17       | 欢迎                                  |
|       | 0x23     | 35       | 读卡成功 取出卡                            |
|       | 0x04     | 4        | 请取出卡                                |
|       | 0x0D     | 13       | 已批准标志                               |
| 复苏州   | 0x05     | 5        | 请仅出示一张卡                             |
|       | 0x03     | 3        | 请仅出示一张卡                             |
|       | 0x27     | 39       | 读取错误请重试                             |
| 无状态消息 | 0x12     | 18       | 谢谢                                  |
|       | 0x15     | 21       | 请使用另一张卡                             |
|       | 0x19     | 25       | 请稍等                                 |
|       | 0x1A     | 26       | 国际卡请刷卡！                             |
| 在线结果  | 0x24     | 36       | 授权中请等待                              |
|       | 0x06     | 6        | 得到正式认可的                             |
|       | 0x09     | 9        | 未批准                                 |
| 离线结果  | 0x07     | 7        | 得到正式认可的                             |
|       | 0x08     | 8        | 未批准                                 |
| 错误状态  | 0x1E     | 30       | 已超时请插卡或刷卡                           |
|       | 0x25     | 37       | 通信失败                                |
|       | 0x26     | 38       | 请插卡或刷卡                              |
|       | 0x0A     | 10       | 已终止                                 |
|       | 0x0B     | 11       | 需要输入 PIN 码 – 请咨询服务员                 |
|       | 0x0C     | 12       | 查看电话                                |
|       | 255      | 不适用      | 自 SDK v1.5 起已弃用，由下方索引 201 至 220 替代。 |

## 来自 EMV Core 的其他消息

自 SDK v1.5 起，EMV Core 使用专用索引发送附加状态消息，而不再使用早期版本中单一的索引 255。您可以使用相应的配置键覆盖每条消息的默认文本。

| 十进制（API） | 默认文本                            | 配置键                           |
| :------- | :------------------------------ | :---------------------------- |
| 201      | 正在加载……                          | `SAT_MSG_LOADING`             |
| 202      | 欢迎                              | `SAT_MSG_READER_IDLE`         |
| 203      | 正在出货                            | `SAT_MSG_VEND_ONLINE`         |
| 204      | 谢谢！总计：X.XX                      | `SAT_MSG_RESULT`              |
| 205      | 拒绝                              | `SAT_MSG_DECLINED`            |
| 206      | 请选择商品                           | `SAT_MSG_SELECT`              |
| 207      | 错误，请重试                          | `SAT_MSG_ERROR`               |
| 208      | 请 / 稍候……                        | `SAT_MSG_DISABLED`            |
| 209      | 未选择，正在取消                        | `SAT_MSG_CANCEL`              |
| 210      | 重新开始                            | `SAT_MSG_SELECTION_TIMEOUT`   |
| 211      | 余额不足                            | `SAT_MSG_LOW_BALANCE`         |
| 212      | 正在更新……                          | `SAT_MSG_BUSY`                |
| 213      | 请插入、刷卡或尝试其他卡片                   | `SAT_MSG_SWIPE_ALL`           |
| 214      | 金额过小                            | `SAT_MSG_MIN_PRICE`           |
| 215      | 已收到余额，请选择商品                     | `SAT_MSG_SELECT_NOCARD`       |
| 216      | 无法识别卡片                          | `SAT_MSG_CARD_NOT_RECOGNIZED` |
| 217      | 正在授权...                         | `SAT_MSG_CLP_AUTHORIZING`     |
| 218      | 已批准，余额：                         | `SAT_MSG_CLP_APPROVED`        |
| 219      | 视情况而定。启动时的固件版本或 IP 地址。          | `SAT_MSG_SYSTEM_INFO`         |
| 220      | 视情况而定。由主机使用 `ShowMessage` 命令设置。 | `SAT_MSG_CUSTOM_MESSAGE`      |

<Note>
  索引 221 至 250 保留供 EMV Core 未来的消息使用。
</Note>
