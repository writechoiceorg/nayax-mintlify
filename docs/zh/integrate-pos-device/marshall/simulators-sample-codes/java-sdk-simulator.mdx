Nayax Marshall SDK 包含一个模拟器，可帮助开发者测试和调试其与 Nayax 支付系统的实现。该模拟器可模拟实际硬件和软件交互的行为，让您无需实体硬件即可验证自动售货流程。

## 模拟器

当 Java SDK 模拟器在终端上启动时，其菜单将如下所示：

<TerminalBlock variant="ios">
  \[0]: 开始售货会话 0 \[1]: 取消会话 \[2]: 关闭会话 0 \[3]: 会话信息 \[4]: 查询余额 \[5]: 查询已打开的会话 \[6]: 查询读取器状态 \[7]: 现金交易 \[8]: 提示 \[9]: 选择品牌 \[10]: 转账自定义交易 ID \[11]: 显示消息 \[12]: 打开套接字 \[13]: 关闭套接字 \[14]: UI 控制 \[15]: 编组解析器 \[q]: 退出
</TerminalBlock>

## 交易流程示例

以下是每种交易类型的流程示例：

1. 常规交易（单一会话）：
   1. 启动会话：使用`[0]: Start session`。
   2. 出示银行卡，等待授权和“谢谢”屏幕。
2. 多会话交易：
   1. 使用`[0]: Start session 0`启动会话。
   2. 出示银行卡，等待授权和“谢谢”屏幕。
   3. 关闭相关会话——在此情况下，使用`[2]: Close Session 0`。
