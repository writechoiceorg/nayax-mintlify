Marshall认证流程包含一系列测试，您（客户）在完成SDK的实现后必须完成这些测试。这些测试旨在确保系统在正式上线前保持稳定，避免出现意外问题。

## 认证流程

当您到达这一阶段时,系统会要求您通知您的 Nayax 集成人员,我们会向您发送与您所使用的流程(Single Session/ Multisession/ Multivend)相匹配的认证测试文件。
该文件包含了您需要执行的测试的说明,以及完成每项测试所需的步骤。正如在集成过程中向您说明的那样,我需要您启用 SDK 的日志记录功能(并且即使在上线后也保持启用,因为这是 Nayax 用于协助您排查上线后可能遇到的问题的主要调试工具),并请您将每个测试结果以不同的文件发送给您的 Nayax 集成人员。
测试过程并不长(假设实现是正确完成的),但它涵盖了大多数(即便不是全部)边界情况。在测试期间,您的 Nayax 集成人员会为您提供指导和协助,直至完成。

认证完成后会发生什么？为什么只有在完成认证后才允许部署至关重要？

一旦所有强制性测试都成功完成，意味着您的 Nayax 集成商已收到所有测试的日志并已批准所有测试，他将通过向您发送正式的认证信函来告知您集成和认证过程已成功完成，并通知分销商，只有到那时分销商才会允许您进行部署。

<Danger>
  重要说明

  该认证有效期为 3 年。为保持合规性，您可能需要在此期限后重新认证，以确保您的 SDK 实现和 Marshall 相关服务继续按预期运行。
</Danger>
