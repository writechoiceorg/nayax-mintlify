Integrating with Spark enables your backend system to control Nayax's payment terminals through secure, API-driven workflows. Nayax provides a structured onboarding process, certification checklist, and production rollout plan to ensure a stable and compliant implementation.

This page outlines how Nayax supports partners through integration and explains the steps involved from start to finish.

## What Nayax Provides

Nayax provides the following resources and materials during the kickoff:

* OpenAPI YAML file containing the full specification of all available endpoints.
* Sample Codes
* Certification checklist to validate your Spark implementation.
* Sandbox credentials, including the API token, Sign Key, and test environment URLs.
* Support channels such as Microsoft Teams or email support from Nayax integration engineers.

<Check>
**OpenAPI**

Use the OpenAPI YAML with tools like Postman or Swagger UI to quickly test your requests during development.
</Check>

## Integration Process

Nayax follows a phased approach to help developers integrate Spark:

1. **Kickoff**: You meet with Nayax to review the integration scope and receive documentation.
2. **Sandbox Setup**: Nayax provisions a test environment and credentials for your team.
3. **Implementation**: You integrate with the Spark API using the provided tools and guides.
4. **Certification**: Nayax reviews your implementation against a test checklist.
5. **Production Deployment**: Once certified, you receive production keys and go live.
