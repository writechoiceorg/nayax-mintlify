title: "Glossary"
description: "Key terms used throughout FisCortina documentation."
---

Use this page as a reference when you encounter an unfamiliar term in the FisCortina guides.

| Term | Definition |
|---|---|
| **3rd Party Server** | The integrator's server that implements the FisCortina API endpoints. |
| **FisCortina MS** | Fiscal Microservice: the Nayax backend component that routes fiscal requests to the integrator. |
| **Integrator** | Any third party connecting with the FisCortina API. |
| **Nayax Device** | Nayax POS terminal enabling cashless in-person payments. |
| **Fiscal Brain** | Nayax backend service that evaluates payment commands and triggers fiscal actions. |
| **Pre-Authorization** | Flow where an initial authorization holds a default amount, settled later at the final amount. |
| **Pre-Selection** | Flow where the product price is known at card presentation (1-step or 2-step). |
| **FiscalId** | Unique fiscalization ID on the integrator's system. Echoed in Void requests. |
| **FiscalRrn** | Fiscal Reference Retrieval Number. Mandatory in the Register response; echoed in Void. |
| **Secret Token** | Pre-shared 66-character key used in StartSession AES encryption. |
