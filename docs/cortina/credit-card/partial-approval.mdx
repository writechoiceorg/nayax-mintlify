This feature lets the payment gateway approve a lower authorization amount if the cardholder does not have enough balance to cover the full requested amount, increasing the approval rate.

Use the partially approved amount for the purchase. It limits the settlement amount.

## How to implement

In the Authorization endpoint response, set the attributes as follows:\
• `Status.Verdict` = Approved\
• `Status.Code` = 0\
• `PaymentInfo.AuthAmount` = Approve amount for pre-authorization.

See the example below:

```json
{"Status": {
	"Verdict": "Approved",
	"Code": 0,},
"PaymentInfo": {
	"AuthAmount": 5}
}
```
