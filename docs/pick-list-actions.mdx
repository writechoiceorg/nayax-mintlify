## Get Pick List from Machine

You can retrieve the pick list associated with a specific machine using the [Get Pick List](ref:get-pick-list) endpoint. See the example request below:

```curl Request
curl --request GET \
     --url https://qa-lynx.nayax.com/operational/v1/machines/{MachineID}/pickList \
     --header 'Authorization: BEARER-TOKEN' \
     --header 'accept: application/json'
```

<Note>
**Path Param**

Change `MachineID` with the actual ID of the machine to retrieve the pick list from.
</Note>

A successful response will return the pick list with all the products available on the machine.

## Update Pick List

You can modify the pick list for a specific machine using the [Update Pick List](ref:update-pick-list) endpoint. This endpoint allows you to update the quantities of products to be restocked or adjust other details in the pick list. See the example request below:

```curl Request
curl --request PUT \
     --url https://qa-lynx.nayax.com/operational/v1/machines/inventory/picklists/update \
     --header 'Authorization: BEARER-TOKEN' \
     --header 'accept: application/json' \
     --header 'Content-Type: application/json' \
     --data '{
       "MachineId": 12345,
       "Products": [
         {
           "MachineProductId": 67890,
           "PickQty": 10
         },
         {
           "MachineProductId": 54321,
           "PickQty": 5
         }
       ]
     }'
```

## Delete Pick List

Using the [Delete Pick List](ref:delete-pick-list) endpoint, you can remove the existing pick list associated with a specific machine. This endpoint helps operators clear outdated or incorrect pick lists to ensure accurate restocking processes.

```curl Request
curl --request DELETE \
     --url https://qa-lynx.nayax.com/operational/v1/machines/{MachineID}/pickList \
     --header 'Authorization: BEARER-TOKEN' \
     --header 'accept: application/json'
```

A successful request will return the confirmation of the deletion of the pick list.
