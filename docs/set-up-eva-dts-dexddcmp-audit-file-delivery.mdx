title: "Set Up EVA DTS DEX/DDCMP Audit File Delivery"
---

To set up DEX file delivery to Amazon SQS, you must have access to **Nayax Core** with the appropriate user permissions. Specifically, the following roles are required:

* **VMO-Manager**
* **DEX Notify - SQS**

Additionally, ensure you have the **Amazon SQS Queue URL**, **Access Key**, and **Secret Key** obtained during the queue setup process.

<Note>
**Prerequisite: Dex Read Schedule Configuration**

These setup instructions assume that the DEX Read schedule has already been configured.
</Note>

## Operator Level Setup

Follow these steps to configure DEX delivery at the operator level in the Nayax Core:

1. Go to **Administration > Operator**.

   <Frame><img src="/images/docs/set-up-eva-dts-dexddcmp-audit-file-delivery/image1.png" /></Frame>
2. Find the Operator you wish to set up.
3. Click the **Search** button.
4. Select the operator to display all tabs and details.
5. Go to the **Dex** tab.
6. Click to open the **Dex Notify** section.

   <Frame><img src="/images/docs/set-up-eva-dts-dexddcmp-audit-file-delivery/image2.png" /></Frame>
7. Set up Dex Transmission to Amazon SQS:
   a. Select the checkbox next to **Amazon SQS** to enable this option.
   b. Enter the SQS credentials (**Access Key**, **Secret Key** and **Queue URL**).
   c. Click the **Validate** button to ensure the Queue credentials are correct.
   d. Enable Encryption (Optional). See the [**Set Up Encryption**](https://developerhub.nayax.com/docs/set-up-encryption-on-amazon-sqs-delivery)  section for details.

   <Frame><img src="/images/docs/set-up-eva-dts-dexddcmp-audit-file-delivery/image3.png" /></Frame>
8. Choose Dex Format and Filter settings:
   a. Ensure the format is set to **No Formatting**.
   b. Select the desired DEX Read Types. Only the selected Dex Origins will be sent to the Queue.

   <Frame><img src="/images/docs/set-up-eva-dts-dexddcmp-audit-file-delivery/image4.png" /></Frame>
9. Click the **Save Information** button to apply the changes.

   <Frame><img src="/images/docs/set-up-eva-dts-dexddcmp-audit-file-delivery/image5.png" /></Frame>

<Warning>
**Operator Defaults vs Machine-Level Configuration**

Setting up DEX delivery at the operator level does not initiate the delivery of DEX files to the queue. Instead, these configurations act as defaults for new machines under the operatorâ€™s hierarchy. You can propagate these settings using the **Apply Dex to** button or perform individual setups at the machine level as explained below.

<Frame><img src="/images/docs/set-up-eva-dts-dexddcmp-audit-file-delivery/image6.png" /></Frame>
</Warning>

## Machine Level Setup

<Note>
**Prerequisite: Dex Read Schedule Configuration**

These setup instructions assume that the DEX Read schedule has already been configured.
</Note>

To configure DEX delivery at the machine level:

1. Go to **Operations > Machines** in the Nayax Core.

   <Frame><img src="/images/docs/set-up-eva-dts-dexddcmp-audit-file-delivery/image7.png" /></Frame>
2. Find the Operator you wish to set up.
3. Click the **Search** button.
4. Select the operator to display all tabs and details.
5. Go to the **Dex** tab.
6. Click to open the **Dex Notify** section.

   <Frame><img src="/images/docs/set-up-eva-dts-dexddcmp-audit-file-delivery/image8.png" /></Frame>
7. Set up Dex Transmission to Amazon SQS:
   a. Select the checkbox next to **Amazon SQS** to enable this option.
   b. Enter the SQS credentials (**Access Key**, **Secret Key** and **Queue URL**).
   c. Click the **Validate** button to ensure the Queue credentials are correct.
   d. Enable Encryption (Optional). See the [**Set Up Encryption**](https://developerhub.nayax.com/docs/set-up-encryption-on-amazon-sqs-delivery)  section for details.

   <Frame><img src="/images/docs/set-up-eva-dts-dexddcmp-audit-file-delivery/image9.png" /></Frame>
8. Choose Dex Format and Filter settings:
   a. Ensure the format is set to **No Formatting**.
   b. Select the desired DEX Read Types. Only the selected Dex Origins will be sent to the Queue.

   <Frame><img src="/images/docs/set-up-eva-dts-dexddcmp-audit-file-delivery/image10.png" /></Frame>
9. Click the **Save** button to apply the changes.

   <Frame><img src="/images/docs/set-up-eva-dts-dexddcmp-audit-file-delivery/image11.png" /></Frame>

## JSON Message Fields Description

The following table describes the fields included in the DEX/DDCMP audit file JSON message:

| **Field Name**         | **Type** | **Description**                              |
| ---------------------- | -------- | -------------------------------------------- |
| **HWSerial**           | `string` | Hardware serial number of the machine.       |
| **MachineDate**        | `string` | Date and time recorded by the machine.       |
| **OperatorIdentifier** | `string` | Unique identifier for the machine operator.  |
| **Origin**             | `string` | Source or origin of the data.                |
| **DexData**            | `string` | DEX protocol data captured from the machine. |
