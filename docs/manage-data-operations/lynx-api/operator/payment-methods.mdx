Payment methods define the transaction types an operator's machines can accept. You can manage these assignments programmatically using the Lynx API.

<Warning>
**Authentication**

Refer to the [Security & Token](/docs/manage-data-operations/lynx-api/security) page to learn how to retrieve your token and authenticate requests.
</Warning>

## PaymentMethodID values

Use the following IDs when creating or updating payment methods:

| PaymentMethodID | Method |
|----------------|--------|
| `1` | Credit Card |
| `2` | NFC |
| `3` | QR |

For the full list, call `GET /v1/lookups/paymentMethods`.

## Create a payment method

```sh
curl --request POST \
     --url "https://qa-lynx.nayax.com/operational/v1/actors/{ActorID}/paymentMethods" \
     --header 'Authorization: Bearer <YOUR_TOKEN>' \
     --header 'Content-Type: application/json' \
     --data '[
       {
         "ActorID": 2009586082,
         "PaymentMethodID": 2,
         "ExtraChargePercentageBit": false,
         "ExtraChargeValue": 0,
         "ExtraChargeBackupValue": 0
       }
     ]'
```

<Warning>
**ActorID required in each array item**

Each object in the request array must include the `ActorID` field matching the actor in the URL path. Omitting it returns `400` with error key `update_actor_payment_multiple_specifications` and message `"The following payment methods dont match the actor id {id}"`.
</Warning>

<Note>
**Duplicate payment methods**

Creating a payment method that already exists on the actor returns `400` with error key `create_actor_payment_not_recognized` and message `"The following payment methods already exist: {id}"`. Retrieve the current list first with `GET /v1/actors/{ActorID}/paymentMethods` before adding.
</Note>

## Delete a payment method

`DELETE /v1/actors/{ActorID}/paymentMethods/{PaymentMethodID}` returns the following on success:

```json
{
  "Ok": true,
  "Message": null,
  "SystemMessage": null,
  "code": null
}
```
