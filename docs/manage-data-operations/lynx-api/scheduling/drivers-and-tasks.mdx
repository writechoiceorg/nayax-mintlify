Drivers are field service users who carry out machine visits. Machine tasks and visit orders define what needs to be done at each machine, combining driver, route, date, and task into a single scheduled work item.

<Warning>
**Elevated permissions required**

Driver and task management endpoints require permissions beyond a standard operator token. See [Error Handling](/docs/manage-data-operations/lynx-api/error-handling) for details.
</Warning>

When creating a driver, provide the following fields:

| Field | Description |
|-------|-------------|
| `UserId` | The Nayax user ID to associate with this driver |
| `OperatorId` | The operator under which the driver is registered |
| `DriverStatusId` | `1` = Active, `2` = Inactive |
| `StartDate` | Driver start date in ISO 8601 format |
| `DriverLicense` | Driver license number |
| `IdNumber` | National ID or employee number |
| `Skills` | Array of skill IDs assigned to this driver |
| `WorkingHours` | Array of working hour slot objects |

Key fields when creating a machine task:

| Field | Description |
|-------|-------------|
| `MachineId` | The machine to schedule a task for |
| `TaskLutId` | The type of task (lookup value) |
| `DriverId` | The driver assigned to this task |
| `ScheduleDate` | When the task is scheduled (ISO 8601) |
| `StatusId` | Task status |
| `GeneratePickList` | `true` to auto-generate a pick list for this visit |
| `TimezoneOffset` | UTC offset in minutes for the scheduled time |
| `Pattern` | Recurrence pattern for recurring tasks |
