The Lynx inventory workflow follows a fixed four-step order: create product groups, create operator products, assign products to machines, then create pick lists. Each step depends on the previous one completing successfully.

## Product groups

Product groups are category containers. Every operator product requires a `ProductGroupID`, so creating at least one group is always the first step.

Create a group:

```http
POST /v1/productGroups
Content-Type: application/json

{
  "ProductGroupName": "Beverages",
  "ProductGroupCode": "BEV",
  "LanguageId": 1
}
```

A successful response returns the new `ProductGroupID`. Save this value for use in the next step. Use `LanguageId: 1` for English.

<Note>
Product groups cannot be deleted via the Lynx API. There is no `DELETE /v1/productGroups` endpoint. Plan your group structure before creation.
</Note>

To list all existing groups, call `GET /v1/productGroups`. To retrieve or update a specific group, use `GET /v1/productGroups/{ProductGroupID}` or `PUT /v1/productGroups/{ProductGroupID}`.

The tax sub-resource endpoints (`/v1/productGroups/{id}/tax`) require elevated permissions beyond the standard operator token. They return 403 with a standard token. Contact Nayax to enable tax permissions before calling those endpoints.

## Operator products

An operator product wraps a Nayax global catalog item with your operator-specific details such as price and group assignment. Before creating a product, find a valid `NayaxProductID` from the global catalog:

```http
GET /v1/nayaxProducts
```

Sandbox test value: `NayaxProductID = 999998535696561`.

Then create the operator product, replacing `{ActorID}` with your operator's ActorID (not a machine ID). Sandbox ActorID: `2009586082`.

```http
POST /v1/operators/{ActorID}/products
Content-Type: application/json

{
  "NayaxProductID": 999998535696561,
  "ProductName": "Cola 330ml",
  "ProductGroupID": 12345,
  "Price": 1.50
}
```

<Warning>
**`NayaxProductID` and operator `ProductID` are different values**

The `NayaxProductID` identifies a product in the Nayax global catalog. When you create an operator product, the API returns a separate operator-specific `ProductID`. All subsequent steps (machine products, pick lists) use this operator `ProductID`, not `NayaxProductID`. Do not use them interchangeably.
</Warning>

To list all products under your operator, call `GET /v1/operators/{ActorID}/products`.

## Machine products

Machine products link your operator products to specific vending machines. Use the operator `ProductID` returned in the previous step, not the `NayaxProductID`.

```http
POST /v1/machines/{MachineID}/products
Content-Type: application/json

{
  "ProductID": 67890,
  "MachineID": 1002529791
}
```

The `MachineID` in the request body must match the `{MachineID}` in the URL path.

To list all products currently mapped to a machine:

```http
GET /v1/machines/{MachineID}/products
```

To update machine-level details such as price or slot number after initial assignment:

```http
PUT /v1/machines/{MachineID}/products/{ProductID}
Content-Type: application/json

{
  "Price": 1.75,
  "SlotNumber": 3
}
```

Read the current machine product state with `GET` before sending an update to avoid overwriting fields unintentionally.

## Pick lists

A pick list is a restock order for a machine. Create one only after the machine products step is complete. You can only include products that are already assigned to the machine.

To create a pick list with specific restock quantities:

```http
POST /v1/machines/{MachineID}/pickList
Content-Type: application/json

{
  "Products": [
    { "ProductID": 67890, "Quantity": 10 },
    { "ProductID": 67891, "Quantity": 5 }
  ]
}
```

`ProductID` is the operator-specific product ID, not `NayaxProductID`.

<Warning>
**Never send an empty `Products` array**

Sending `{"Products": []}` causes a 500 Internal Server Error. To create a pick list with no items, omit the `Products` key entirely and send `{}` instead.
</Warning>

The `POST` response body is empty. Use `GET /v1/machines/{MachineID}/pickList` to confirm the pick list was created and inspect its contents.

If you use curl with an empty body, add `-H "Content-Length: 0"`. Without it, curl omits the header and the server returns 411 Length Required.

The API maintains one active pick list per machine. Posting a new pick list replaces the existing one.
