Authentication using a bearer token is required for every Lynx API request. Tokens are static credentials retrieved from Nayax Core; you do not generate them via the API.

## Access Token

An access token is a security credential used to authenticate API requests. Each request to the Lynx API must include a valid access token in the `Authorization` header.

<Warning>
**Keep your Tokens Safe**

It is important to avoid sharing your secret tokens in public places like GitHub or Bitbucket since it can allow malicious API calls.
</Warning>

Access tokens can be found through the Nayax Back Office (Nayax Core). Follow these steps to retrieve an access token:

1. **Log in to Nayax Core** using your username and password.
2. **Navigate to Account Settings** by clicking on your username at the top right corner, and selecting the **Account Settings** option.

    <Frame><img src="/images/docs/security/image1.png" /></Frame>

3. Access the **Security and Login** Tab.

    <Frame><img src="/images/docs/security/image2.png" /></Frame>

4. Navigate down to the **User Tokens** section.

    <Frame><img src="/images/docs/security/image3.png" /></Frame>

5. Click the **Show Token** button next to an existing token to show it, and copy the token for use in API calls.

    <Frame><img src="/images/docs/security/image4.png" /></Frame>

## Using the Token in API Requests

Include the access token in the `Authorization` header for each API request. Here’s an example `curl` command:

```sh
curl -X GET "https://lynx.nayax.com/operational/api/v1/devices" \
-H "Authorization: Bearer <YOUR_ACCESS_TOKEN>" \
-H "Content-Type: application/json"
```

* Replace `<YOUR_ACCESS_TOKEN>` with your actual token retrieved from Nayax Core.

## Encryption Key Endpoints

The Lynx API includes endpoints for managing encryption keys and decrypting messages:

* `GET /v1/actors/{actorID}/EncryptionKeys` — retrieve encryption keys for an actor
* `PUT /v1/actors/GenerateEncKey` — generate a new encryption key for an actor
* `PUT /v1/actors/DecryptionMessageByVer` — decrypt a message using a specific encryption version

<Warning>
**Elevated Permissions Required**

These endpoints require permissions beyond those granted to a standard operator token. Requests made with a standard bearer token will return `403 Insufficient Permissions`. Contact your Nayax account manager or distributor to have the necessary permissions added to your account before using these endpoints.
</Warning>
