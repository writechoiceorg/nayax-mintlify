Several Lynx API fields, including `CountryID`, `CurrencyID`, `ActorTypeID`, and `TimeZoneKey`, require specific numeric identifiers from the Nayax system. Use the reference data endpoints below to retrieve the valid values before making create or update requests.

<Warning>
**CountryID is not the ISO numeric code**

The `CountryID` values returned by `GET /v1/countries` are internal Nayax identifiers, not ISO 3166 numeric codes. For example, the United States has `CountryID: 225` in the lookup table, but `840` (the ISO code) appears in some actor fields. When passing `CountryID` to lookup endpoints for states, cities, or regions, always use the internal ID from `GET /v1/countries`, not the ISO code.
</Warning>
