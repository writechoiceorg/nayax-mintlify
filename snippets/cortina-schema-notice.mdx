## API Schema Modification Notice

Nayax reserves the right, at its sole discretion, to add new fields to the JSON request and response schemas of its APIs from time to time. Additions may support new functionality, enhanced data reporting, or regulatory compliance. Nayax provides at least fourteen (14) calendar days' advance notice before introducing any new field, delivered through the Developer Zone, release notes, email, or other reasonable means.

No existing field in a published JSON schema will be removed, renamed, or repurposed in a way that breaks backward compatibility. Design your integration to safely ignore any additional, previously undocumented fields it does not recognize, so it stays compatible with future schema additions.

This notice period does not apply to changes required to remediate a security vulnerability, comply with a legal or regulatory obligation, or address an urgent operational issue. In those cases, Nayax will use commercially reasonable efforts to notify affected customers as promptly as practicable.
