This section provides detailed reference documentation for all API methods available in the Spark integration. Each method includes its purpose, request structure, response structure, and key fields. Use this reference when building or maintaining your integration with Spark.

All requests must follow the security and authentication guidelines outlined on the [Security & Authentication](/docs/security-authentication) page. Most endpoints require signed headers and valid Spark transaction identifiers.
